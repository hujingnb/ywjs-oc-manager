# 用户手册

> 平台管理员 / 企业管理员 / 企业成员三类角色在 manager 前端能做什么。按角色分章节，每章给出页面入口、典型操作步骤与权限边界。

---

## 登录

访问 manager 部署地址（本地默认 `http://localhost:5173`），进入登录页（路由 `/login`）。

**登录表单字段**（自上而下三个输入框）：

- **企业标识**：平台管理员留空；企业用户填写所属企业的 `code`（如 `test-org`）。占位提示为「企业用户填写，平台管理员留空」。
- **账号**：登录账号。
- **密码**：登录密码；右侧「眼睛」图标可切换明文/密文显示。

页面不提供自助注册入口。若部署侧开启了人机校验，表单会常驻一枚验证码组件，加载后自动完成校验，无需手动点击。

**界面语言切换（中/英）**：登录卡片右上角有语言选择器，可在 **简体中文** 与 **English** 间切换（当前仅支持这两种语言）。默认语言优先取本机上次选择（localStorage），否则采用平台默认语言，最终兜底为英文。登录页切换会记住在本机（localStorage），跨刷新/跨会话在本机持久，但不同步到账号偏好；登录后可在顶栏语言选择器再次切换，此时才会持久化到账号偏好并跟随账号跨设备生效。

提交后系统颁发 access token + refresh token；access token 过期时前端自动用 refresh token 续签，无需手动重新登录。

**重置密码**：不提供自助找回；请联系企业管理员或平台管理员在成员管理页重置。

**登录后跳转**：进入角色首页（`RoleAwareHome`），按角色分流：

- 平台管理员（`platform_admin`）→ 平台控制台 `/console`。
- 企业管理员（`org_admin`）→ 企业控制台 `/org-console`。
- 企业成员（`org_member`）→ 其名下实例概览页；无实例时进入实例空态页 `/apps/empty`。

（若登录前因未授权被拦截，会带上 `redirect` 参数，登录成功后回跳到原目标页。）

## 1. 平台管理员

平台管理员（`platform_admin`）管理整个平台，拥有所有资源的最高访问权限。详细权限规则参见 [`docs/product-design.md`](product-design.md)。

### 1.1 平台总览

页面路径：`/console`（登录后默认落地页；旧路径 `/platform/dashboard`、`/dashboard` 均重定向至此）。

页面顶部为一条统计卡片区，下方为图表 Tab 区。

**统计卡片（六张）**：

| 指标 | 说明 |
|---|---|
| 企业数 | 已创建企业总数 |
| 成员数 | 所有企业成员总数（副标题标注「不含平台管理员」） |
| 实例数 | 所有实例总数 |
| 运行中 | 状态为 running 的实例数 |
| 异常 | 状态为 error 的实例数 |
| 今日 Token | 今日全平台 Token 消耗，直查 new-api 实时用量；数据未就绪时显示 `—` 并标注「加载中…」或「不可用」 |

> 统计卡片已不再展示「总余额」；余额改由各企业行 / 充值页按需实时查询。

**图表 Tab（三个）**：

- **Token 趋势**：近 7 天平台 Token 消耗折线图；用量服务不可用时显示「用量服务不可用」。
- **各企业用量**：各企业 Token 用量横向柱状图；无数据时显示「暂无数据」。
- **实例状态**：实例状态占比饼图（运行中 / 停止 / 异常）；平台数据不可用时显示「平台数据不可用」。

指标与图表均为实时查询，Token 相关数据按分钟级自动刷新。

**权限边界**：仅平台管理员可访问；`/console` 是平台管理员的默认首页。

### 1.2 企业管理

页面路径：`/organizations`

**查看企业列表**，每行展示：名称（备注作为副标题）、企业标识（`code`）、状态、联系人、电话、预警阈值（`%`）、实例上限（未设置显示「不限」）、知识库空间（GB）、当前余额（对每个企业并发实时查询 new-api，未加载时显示 `…`）。

**创建企业**：

1. 点击右上角「新增企业」展开表单卡片。
2. 必填字段：
   - 名称
   - 企业标识（`code`，登录命名空间，创建后不可修改）
   - 管理员账号（`admin_username`）
   - 管理员姓名（`admin_display_name`）
   - 管理员密码（`admin_password`）
3. 可选字段：联系人、联系电话、余额预警阈值（0–100 %）、实例数量上限（留空 = 不限制）、企业知识库空间（GB）、个人知识库空间（GB，作为该企业新建实例的默认个人库上限，仅对之后新建实例生效）、备注、可用助手版本（多选，可留空）。
4. 点击「保存」提交；系统自动在 new-api 创建对应账号。

> 表单中不再设置「可用模型」，也不在此配置企业 AI 人设；企业可用能力通过「可用助手版本」多选授予。

**编辑企业**：点击对应行「编辑」，以编辑模式打开同一表单。可修改名称、联系人、电话、预警阈值、实例上限、知识库空间、个人知识库空间、备注、可用助手版本；企业标识只读展示，编辑模式不含管理员账号字段。

**启用 / 禁用企业**：在对应行点击「禁用」或「启用」（两个按钮按当前状态互斥显示）。切换直接生效，无二次确认弹框。禁用后该企业成员无法登录。

**复制企业信息**：点击「复制信息」，将企业标识、名称、管理员账号及密码提示（明文不保存）按固定格式写入剪贴板，便于转发给企业管理员；结果以页面内文字反馈。

**充值 Token Credit（列表内弹框）**：

1. 点击目标企业行的「充值」，弹出「企业充值」弹框。
2. 弹框顶部展示当前余额（剩余 / 已用，直查 new-api，不缓存）。
3. 填写充值金额（正整数）和可选备注。
4. 点击「确认充值」提交；金额非正整数时在弹框内提示，充值结果以文字反馈。

> 该弹框为直接确认，不要求输入企业名称二次校验。充值为内部记账操作，不涉及在线支付流程。

**查看充值记录**：点击对应行「充值记录」，弹出记录弹窗，顶部概况展示「累计充值金额」和「当前剩余金额」，下方表格列出时间、金额、备注、状态（成功 / 失败）、操作人。

**独立充值页**：另有独立充值页 `/platform/organizations/:orgId/recharge`（`RechargePage`），布局为充值表单 + 充值历史两张卡片。该页的确认走 `ConfirmActionModal` 二次确认，需在弹框中输入目标企业名称才能提交，历史表格额外展示错误信息列。当前 UI 未提供直达该页的入口按钮，主要通过直接访问该路由进入（页内「返回企业列表」链接回到 `/organizations`）。

**权限边界**：企业管理与充值相关页面仅平台管理员可访问。

### 1.3 权限管理

页面路径：`/platform/permissions`

**只读**的权限矩阵说明页，用于对照三类角色（平台管理员 / 企业管理员 / 企业成员）在各功能模块下的可操作范围。页面不调用任何接口、无任何可执行操作，纯静态展示。

页首有图例：`✅` 可操作（无条件）、`❌` 无权限、`🟡` 有条件（本企业 / 仅自己）。矩阵单元格中的有条件取值还包含「本企业全部」，用于区分「本企业范围内全部」与「仅自己」。

矩阵按功能模块分组为多张表格，涵盖：企业管理、成员管理、应用实例、运行时操作、渠道（Channel）、知识库、助手版本、任务看板（Kanban）、Cron 任务、用量、审计日志、充值记录、运行时节点、平台总览、模型列表、后台任务（Jobs）、工作区、资源指标。每张表按「操作 / 平台管理员 / 企业管理员 / 企业成员」四列列出各角色权限。

> 此页仅呈现权限设计，实际鉴权以后端 `internal/auth/authorizer.go` 为准。

**权限边界**：仅平台管理员可访问。

### 1.4 助手版本

页面路径：`/assistant-versions`

平台管理员维护「助手版本」目录：一个助手版本是一套可复用的实例运行配置（镜像 + 内置提示词 + 主模型 + 智能路由 + 行业知识库 + Skill）。企业创建 / 编辑时通过「可用助手版本」多选授予（见 §1.2），实例落地时套用某个版本。

**查看版本列表**：表格列展示——名称（描述作为副标题）、镜像（把 `image_id` 映射为可读镜像名）、主模型、修订号（`r{n}`）、Skill 数、操作（编辑 / 删除）。右上角「新增版本」打开表单。

**新建 / 编辑版本**（列表下方展开表单卡片，保存 / 取消按钮固定在卡片顶部）：

| 字段 | 必填 | 说明 |
|---|---|---|
| 名称 | 是 | 版本名称，需唯一 |
| 使用镜像 | 是 | 从运行时镜像列表中选择 Hermes 镜像 |
| 描述 | 否 | 版本用途说明 |
| 内置提示词 | 是 | 该版本的系统提示词，注入容器 `SOUL.md` 的**版本层**；可填写助手人设、行为规则等 |
| 主模型 | 是 | 主对话模型（可搜索） |
| 智能路由 | 否 | 8 个 auxiliary 槽位——图像识别、上下文压缩、网页提取、会话搜索、标题生成、智能审批、技能检索、MCP 路由；每个槽位可单独选模型，留空则该能力走主模型 |
| 行业知识库 | 否 | 多选，见下方「行业知识库关联」 |
| Skill 列表 | 否 | 从技能市场为版本配置 skill，见下方「Skill 配置」 |

必填项（名称、内置提示词、镜像、主模型）齐备且镜像 / 模型列表加载正常时「保存」才可点。

> **关于人设**：助手版本里的「内置提示词」是三层人设拼接（平台默认 → 企业 → 实例）中的**版本层**（属平台默认侧）。企业侧不再单独配置人设、也没有独立的人设页面，人设能力由助手版本承载；最终注入容器的 system prompt 由「平台默认 → 企业 → 实例」三层在容器内（`SOUL.md`）拼接。

**行业知识库关联**：在「行业知识库」多选框选择一个或多个平台级行业库（见 §1.5）。对应实例运行时执行 `oc-kb search` 时，除实例知识库、企业知识库外还会检索这些行业库，每个关联库单独返回最多 `top_k` 条结果。表单内常驻一条黄色提示：关联越多，回答要处理的参考内容越多，速度和费用都可能增加，建议只选当前版本真正需要的行业库。行业库关联保存后即时生效，不改变版本修订号。

**Skill 配置**：仅在**编辑态**可配置 Skill（新建态需先保存版本拿到 ID，表单会提示「保存版本后可配置 skill」，创建成功后自动切到编辑态）。编辑态下：

- 已配 Skill 以「名称 + `v{版本}`」列出，每项可「删除」。
- 通过技能市场浏览器从**平台库或外部源（如 ClawHub）**选取 skill 并指定版本后「添加」；已配同名 skill 会被去重、不可重复添加。
- 添加 / 删除均即时调用后端生效并刷新本地列表。

**删除版本**：点击行内「删除」弹出 `ConfirmActionModal` 二次确认（提示「确定删除版本『名称』？删除后不可恢复」，直接确认、无需输入版本名）。版本仍被企业或实例引用时后端返回 `409`，错误信息就地展示，需先解除引用再删。

**权限边界**：仅平台管理员可访问与操作。

### 1.5 行业知识库

页面路径：`/platform/industry-knowledge`

平台管理员维护平台级通用行业资料库。行业名称在未删除行业库中唯一。行业知识库不属于任何企业，是否被某实例检索取决于助手版本的行业知识库关联（见 §1.4）。

**行业库列表**（左侧表格）列：行业名称、文件数、更新时间、操作。工具栏含：行业名称搜索框、「接口文档」按钮、「新建行业库」按钮。

行业库行内操作：

- **文件**：选中该库，右侧展开其文件管理卡片（选中行高亮为主色）。
- **RAGFlow 信息**：查看该库对应 RAGFlow dataset 的运维信息（仅具备 RAGFlow dataset 运维权限的平台管理员可见；后端仍做最终校验）。
- **重命名**：通过浏览器 `prompt` 输入新行业名称。
- **删除**：浏览器原生 `confirm` 确认（「确认删除行业库『名称』？」）。行业库仍被任一未删除助手版本引用时，后端拒绝删除，需先在助手版本中移除关联。

**文件管理**（选中某行业库后）：卡片头部展示允许的文件类型与单文件大小上限，并提供「清空文件」「上传文件」入口；正文顶部常驻提示「同名文件会覆盖当前行业库内的旧文件」。

- **上传文件**：原生文件选择框（支持多选）。文件写入该行业库对应 RAGFlow dataset 并触发解析；同名文件覆盖旧文件。上传进度经统一进度弹窗汇总。
- **文件筛选**：文件名搜索、解析状态下拉（全部状态 / 各解析态）、创建日期范围；文件表格为后端远程分页（每页 10 / 20 / 50 / 100）。
- **文件列表列**：文件名称、大小、类型、解析状态（状态标签 + 解析中显示百分比进度 + 失败显示错误原因）、创建时间、操作。
- **文件操作**：下载；重解析（仅解析 `failed` / `stopped` 的文件显示）；删除（浏览器原生 `confirm` 确认「确认删除 {名称}？」）。
- **清空文件**：清空当前行业库全部文件，弹 `ConfirmActionModal` 并**要求输入行业库名称**二次校验；行业库记录与助手版本关联保留，操作不可撤销。

**新建行业库**：点击「新建行业库」弹出对话框，填写行业名称后「确认创建」。

**外部上传接口文档**：点击工具栏「接口文档」弹出说明框。页面从 manager 当前配置读取 `industry_knowledge.upload_token` 并注入弹框正文、curl 示例与「复制 Markdown」内容，便于交付给外部服务方。token 为空时外部上传入口禁用（弹框显示「未配置，外部上传入口禁用」）；token 仅含空白字符时 manager 会启动失败。

该外部接口供外部商业知识库服务方直接把资料写入平台行业库：按 `industry_name` 自动创建或复用行业库，同一行业库内同名文件覆盖旧文件。

接口地址：

```text
POST /api/v1/external/industry-knowledge/files
```

鉴权 Header：

```text
X-OC-Industry-Knowledge-Token: <industry_knowledge.upload_token>
```

表单字段（`multipart/form-data`）：

| 字段 | 必填 | 说明 |
|---|---|---|
| `industry_name` | 是 | 行业名称。不存在时自动创建行业库；未删除行业库中名称唯一。 |
| `file` | 是 | 上传文件。同一行业库内同名文件会覆盖旧文件。 |

curl 调用示例：

```bash
curl -i \
  -H "X-OC-Industry-Knowledge-Token: <页面显示的实际 upload_token>" \
  -F "industry_name=保险" \
  -F "file=@./policy.pdf;type=application/pdf" \
  https://<manager-domain>/api/v1/external/industry-knowledge/files
```

返回码：

| 状态码 | 说明 |
|---|---|
| `202` | 上传成功，文件进入 RAGFlow 解析队列。 |
| `400` | 参数缺失、行业名称为空或请求体格式错误。 |
| `401` | 缺少或错误的 `X-OC-Industry-Knowledge-Token`。 |
| `413` | 文件大小超过平台上传限制。 |

上传成功后通常先返回 `parse_status=queued`，解析完成后才能稳定参与检索。外部上传只负责写入行业库；实例是否检索该行业库仍由助手版本的行业知识库关联决定；每个关联行业库检索时单独召回最多 `top_k` 条结果，关联过多会增加上下文长度和响应成本。

**权限边界**：仅平台管理员可访问与操作。

### 1.6 平台技能库

页面路径：`/platform/skills`

平台管理员维护「平台技能库」（skill 市场的平台库来源）。skill 可被助手版本从市场选取配置（见 §1.4 的「Skill 配置」）。页面分「上传区」与「平台技能列表」两张卡片。

**上传 skill**（上传区）支持两种方式，前端会在浏览器内校验 SKILL.md 的 frontmatter 并打包成**扁平 tar**（SKILL.md 位于归档根）后经 multipart 上传，`name` / `description` 自动取自 frontmatter：

- **粘贴 Markdown**：在文本域粘贴 SKILL.md 全文，需含 `---` 包裹的 frontmatter（至少含 `name`，`description` 可选）。提供「填充示例」按钮与格式示例。
- **上传文件夹**：选择 skill 自身的文件夹（其中需直接包含 `SKILL.md`，不要选它的上层目录）。子目录与附属文件（如 `scripts/`、`assets/`、`reference.md`）原样保留；技能名取自 `SKILL.md` 的 `name` 字段而非文件夹名，上传时自动剥掉最外层目录按扁平结构打包。

上传区还会实时展示解析预览（成功显示「识别到技能：名称 — 描述」，失败显示红色错误），并要求：

- **版本号**（必填）：格式校验为 `x.x.x` 或 `x.x`（各段纯数字，正则 `^\d+\.\d+(\.\d+)?$`）；格式非法时内联提示「版本号必须为 x.x.x 或 x.x 格式」。前端校验仅为即时反馈，后端才是权威防线（curl / 旧客户端仍会被后端拦截）。
- **描述**：默认取自 frontmatter 的 `description`，可手动修改。

仅当解析成功拿到技能名、版本号已填且格式合法、且不在上传中时「上传」按钮才可点。

**平台技能列表**（列表卡片，表格）列：名称（描述作为副标题）、版本、文件大小、操作（删除）。删除通过弹窗二次确认（「确定删除 skill『名称 版本』？删除后不可恢复」）。

**权限边界**：仅平台管理员可访问与操作。

### 1.7 定制技能工单

页面路径：`/platform/custom-skills`（工单详情 `/skill-tickets/:id`）

企业管理员 / 成员按需求提交定制技能工单，平台侧人工制作后以 `custom` 来源定向交付。平台管理员在此集中处理全部工单。

**工单列表**：顶部提供组织下拉（全部组织 / 具体企业）、状态下拉（全部 / 待处理 / 制作中 / 已交付 / 已拒绝）、按标题关键词过滤。表格列：标题、提交者（角色显示为管理员 / 成员）、状态（彩色标签）、报价（`¥` 元）。整行可点击（含键盘回车 / 空格）进入工单详情。

**工单状态流转**：待处理（`pending`）→ 制作中（`processing`）→ 已交付（`delivered`）；`pending` / `processing` 可「拒绝」转为已拒绝（`rejected`），`rejected` 可「重新受理」回到 `pending`。

**工单详情（平台管理员操作，按状态显示）**：

- **报价**：`pending` / `processing` 状态下可编辑报价（以元输入，保存为分），「保存报价」。
- **开始制作**（`pending`）：接单，转入 `processing`。
- **交付**（`processing`）/ **迭代交付**（`delivered`）：打开交付弹框，上传 skill（粘贴 Markdown 或上传文件夹，与 §1.6 相同的扁平 tar 打包），并设置**可见范围**（一个或多个「企业 + 受众」目标；受众可选：整企业 `all_org` / 仅企业管理员 `org_admins` / 仅申请人 `requester_only`）。默认目标依申请人角色：管理员申请默认「仅企业管理员」，成员申请默认「整企业」。
- **编辑可见范围**（`delivered`）：不重新上传，仅调整交付目标的企业与受众。
- **拒绝**（`pending` / `processing`）：弹框填写拒绝原因，转入 `rejected`；拒绝原因会在详情页展示。
- **重新受理**（`rejected`）：回到 `pending`。

详情页还展示提交信息（提交者 / 所属企业）、报价、可见范围与工单对话（对话区定时刷新模拟实时消息）。

**权限边界**：工单处理（开始制作 / 交付 / 编辑可见范围 / 拒绝 / 重新受理 / 报价）仅平台管理员可执行；企业侧用户在自己视角查看并在交付后前往安装。

### 1.8 web 发布

平台侧涉及两页：web 发布开通配置与已发布站点。二者均为「平台管理员 + 企业管理员」可访问，平台管理员可跨企业操作，本节写平台管理员视角。

#### web 发布开通配置（`/platform/web-publish-config`）

平台管理员通过顶部企业选择器（可搜索）跨企业切换；选中企业后展示配置参数、开通 / 停用、证书状态三块。

**配置参数**：

| 字段 | 必填 | 说明 |
|---|---|---|
| 根域名 | 是 | 企业所有站点子域名的根域名，如 `apps.example.com` |
| DNS 服务商 | 是 | 当前仅开放阿里云 DNS（`alidns`）；华为云 / 腾讯云 / 移动云通配记录尚未实现、暂不在下拉暴露；dev 自签模式下额外提供「本地调试（`local`）」占位项 |
| 站点存活天数 | 否 | 留空 / `<=0` 时后端默认 7 天 |
| 最大站点数 | 否 | 留空 / `<=0` 时后端默认 20 |
| 凭证 | — | 依所选 DNS 服务商展示：Access Key ID、Access Key Secret（华为云额外需「地域」）。凭证为**只写**，保存响应不回填；留空表示保留已有加密凭证 |

根域名与 DNS 服务商非空时「保存配置」才可点；保存成功后清空凭证输入框。

**开通 / 停用**（仅平台管理员可见可操作）：展示当前开通状态（已开通 / 未开通）、`provisioning_status` 及失败时的 `provisioning_message`。

- **开通 Web 发布**：调用 `enable`，触发后台 provisioning 任务；已开通时仍可重新触发。
- **停用 Web 发布**：二次确认弹框后置状态为 `disabled`，配置数据与站点数据保留。

**证书状态面板**（`WebPublishCertPanel`）：展示通配域名、证书状态、证书到期时间、最近首签时间、最近续签时间、失败原因。平台管理员可点击「重试签发 / 续签」；企业管理员在此仅只读。

#### 已发布站点（`/published-sites`）

平台管理员可通过组织选择器跨企业查看某企业的已发布站点，并重试证书。

**站点列表**列：URL（点击在新标签页打开）、发布者（显示名优先，回退登录名）、状态（`active` / `disabled` / `expired`，彩色徽章）、到期时间、大小、操作。

**站点操作**：

- **续期**：所有状态可用，直接提交。
- **下线**：仅 `active` 站点显示，点击弹 `ConfirmActionModal` 二次确认（提示中含站点 URL）后调用下线接口。

页面同样内嵌证书状态面板：平台管理员可重试证书签发 / 续签，企业管理员只读。

**权限边界**：平台管理员可跨企业配置、开通 / 停用、重试证书、查看所有企业站点；企业管理员仅限本企业且无开通 / 停用入口、证书只读。

### 1.9 审计日志（平台视角）

页面路径：`/audit-logs`

平台管理员在此跨企业查看组织级审计日志。工具栏提供企业下拉框切换目标企业（企业管理员访问同一页时无下拉框，锁定为自身企业）。

列表列：

- **时间**：操作发生时间。
- **操作者**：操作者显示名（缺失时回退为截短的 UUID 或本地化角色名），下方副文为角色标签；悬停显示完整 UUID；`system` 系统操作直接展示角色。已删除主体追加「已删除」徽章。
- **资源**：被操作资源名（缺名时展示资源 ID），副文为资源类型标签。
- **操作**：按（资源类型, 动作）本地化后的操作名。
- **详情**：结构化操作详情（如充值金额、级联删除数量、渠道认证、API 调用端点等），无结构化详情时回退到旧行冻结文本，均无则显示「—」。
- **结果**：成功 / 失败 / 部分（`partial`）彩色标签；失败时下方附错误信息。

**权限边界**：平台管理员和企业管理员均可访问；企业管理员只能查看自身企业日志。普通成员在此页无权限，只能在实例详情的审计 tab 查看自己实例的审计。

### 1.10 用量（平台视角）

页面路径：`/usage`

平台管理员看到四个 tab（默认落在「平台」）：

- **企业**：企业下拉框（可搜索）选定目标企业后展示企业维度汇总用量。
- **成员**：先选企业、再选成员，查看单个成员的用量明细。
- **实例**：先选企业、再选实例，查看单个实例的用量明细；所选实例尚未绑定 new-api key 时提示未绑定、不发起查询。
- **平台**：全平台汇总用量。

各维度均直查 new-api 实时数据。企业 / 成员 / 实例三个 tab 对平台管理员都带企业下拉框；成员、实例维度需先选定企业。

**权限边界**：tab 集合与查询范围按角色裁剪，四 tab 仅平台管理员可见。

### 1.11 成员 / 实例（跨企业查看）

平台管理员通过 `/members` 与 `/apps` 两页跨企业查看成员与实例。两页工具栏都有企业下拉框切换目标企业。实例详情各 tab 的操作详见 §2.3。

**成员列表（`/members`）**：列展示用户名、显示名、角色、状态、实例（有活跃实例时为跳转链接，否则显示「无实例」标签）。平台管理员在此页对成员为**只读**：不提供创建 / 启用 / 禁用 / 重置密码 / 删除等写操作（这些入口仅对本企业管理员显示）。

平台管理员唯一可执行的写操作是为**尚无活跃实例**的成员补建实例：点击该成员行的「创建新实例」展开表单，填写实例名并选择**助手版本**（从该企业的可用助手版本 allowlist 过滤，必选），提交后后台异步初始化容器，页面展示新实例名与 Job ID。

> 补建实例已按助手版本模型收敛：表单只选「助手版本」，不再单独选择人设模式或模型（人设 / 模型 / 路由等由所选助手版本承载，见 §1.4）。

**实例列表（`/apps`）**：列展示实例名（跳转详情）、状态（绑定的助手版本已更新但尚未生效时附「需重启」提示标签）、API key 状态、操作。平台管理员可对实例执行：

- **重启** / **停止**：直接提交运行时操作（此列表页不弹二次确认）。
- **删除**：弹 `ConfirmActionModal`，需输入实例名二次校验后才提交。

**权限边界**：平台管理员可跨企业查看全部成员与实例；成员写操作限本企业管理员；实例管理操作按 `canManageApp` 校验（平台管理员对全部实例可管理）。

## 2. 企业管理员

企业管理员（`org_admin`）负责本企业的成员、实例、渠道、知识库与用量管理。登录后进入企业控制台 `/org-console`。企业管理员对本企业内所有实例拥有管理权限（等同实例 owner 的写权限）。

### 2.1 企业控制台

页面路径：`/org-console`（仅企业管理员可见）

登录后的默认首页，是一个只读数据看板，不含写操作。

**统计条**（6 张卡片）：成员数、实例数、运行中（实例）、异常（实例）、当前余额、今日 Token。其中「当前余额」实时查询 new-api（非本地缓存），「今日 Token」按本地日期从近 7 天用量序列汇总。

**图表区**（两个 tab）：

- **用量趋势**：近 7 天本企业按日聚合的 Token 用量折线图。
- **实例状态**：本企业实例按 运行中 / 已停止 / 异常 三类的占比饼图。

### 2.2 成员管理

页面路径：`/members`（列表）、`/members/new`（一站式开通）

**查看成员列表**：展示用户名、显示名、角色、状态、实例（有活跃实例时为跳转详情的链接，否则显示「无实例」标签）。平台管理员在本页顶部有组织选择器切换查看对象，但对成员为只读（见权限边界）。

**创建成员并初始化实例**（`/members/new`）：

点击工具栏「创建并初始化」进入专用表单，一步完成成员账号与配套实例的创建。

账号信息：

| 字段 | 说明 |
|---|---|
| 用户名 | 同一企业下唯一 |
| 显示名 | UI 展示用 |
| 初始密码 | 至少 8 位 |
| 角色 | 企业成员 / 企业管理员 |

实例信息：

| 字段 | 说明 |
|---|---|
| 实例名 | 必填 |
| 助手版本 | 必选，从本企业允许使用的助手版本（组织 allowlist ∩ 版本目录）中选择 |

> 开通表单不再包含「人设模式 / 模型 / 实例 prompt」等字段——模型与人设已收敛到「助手版本」，由平台侧在版本内固化。渠道类型默认 `wechat`，绑定在实例详情的渠道 tab 完成。

提交后后台 worker 异步初始化容器，成功后提示并跳转回 `/members`，可在列表继续观察实例状态。

**仅创建账号（不初始化实例）**：回到 `/members`，点击工具栏「新增成员」展开表单，填写用户名、显示名、初始密码、角色。此方式只建账号、不建实例。

**为已有成员补建实例**：对尚无活跃实例的成员，操作列出现「创建实例」按钮，展开表单填写实例名 + 助手版本后提交。该入口仅在「当前账号有补建权限且该成员无活跃实例」时出现。

**禁用 / 启用成员**：操作列「禁用」「启用」按当前状态互斥显示，点击直接提交，无二次确认。禁用后成员无法登录。

**重置密码**：

1. 点击「重置密码」，弹出 `window.prompt` 输入新密码（至少 8 位，不满足则不进入确认）。
2. 弹出「确认重置」弹框，需输入该成员的登录名确认。
3. 确认后原密码立即失效。

**删除成员**：点击「删除」，弹出确认弹框（说明将禁用账号并提交名下实例的删除任务，不可撤销），点击「确认删除」执行；此弹框无需输入成员名。后台 worker 停止并删除实例容器、注销 API key。当前登录用户不能删除自身（入口隐藏）。

**权限边界**：写操作（新增 / 启停 / 重置密码 / 删除）仅本企业管理员可执行；平台管理员在本页对成员为只读，但可跨企业执行「创建实例」补建。补建实例：平台管理员跨企业可用，企业管理员仅限本企业。

### 2.3 实例列表与快速操作

页面路径：`/apps`

展示本企业所有实例，列包含：实例名（链接跳转详情）、状态、API key 状态、操作。状态列在「助手版本已更新但实例尚未生效」（`version_synced=false`）时附加带 tooltip 的「需重启」标签。

工具栏提供「创建成员」快捷入口（跳转 `/members/new`，仅本企业管理员可见）；平台管理员另有组织选择器。

操作列（仅对有实例管理权限的用户显示）：

- **重启**：点击直接提交重启任务，无二次确认。
- **停止**：点击直接提交停止任务，无二次确认。
- **删除**：弹出确认弹框，需输入实例名确认后提交。

**权限边界**：企业管理员可管理本企业所有实例；企业成员只能看到自己的实例（同样拥有对自有实例的管理权限）。

### 2.4 实例详情

页面路径：`/apps/:appId`（无子路径时默认跳转 `/apps/:appId/overview`）

详情页顶部为实例名与状态徽标，下方 tab 依次为：**对话、概览、看板、定时任务、运行时、渠道、实例知识库、技能、工作目录、审计**。

- **运行时（runtime）tab 仅平台管理员可见**：企业管理员与企业成员看不到该 tab（它承载容器 / 镜像 / 资源趋势等基础设施信息）。企业管理员的启动 / 停止 / 重启 / 删除操作通过实例列表操作列、以及概览等页面内的「立即重启」按钮完成。
- 企业成员进入详情页时顶部 tab 导航隐藏（其已通过左侧菜单直达实例能力）。

各 tab 的写操作普遍受实例管理权限 `canManageApp` 约束：平台管理员对任意实例、企业管理员对本企业实例、企业成员对自有实例可写；其余为只读。

#### 对话 tab（`/apps/:appId/conversations`）

与实例内 Hermes 助手进行会话管理与流式对话，是详情页排在首位的高频入口。

- **布局**：左侧会话列表（每条显示来源标签与标题，标题缺失时回退会话 ID），右侧为消息区 + 输入区。
- **新建会话**：点击「新建」直接创建。
- **发送消息 / 流式回复**：输入后发送，助手回复以 SSE 流式增量返回；发送中输入框与发送按钮禁用。首条消息后会尝试自动命名会话（失败静默，不影响使用）。
- **发送队列**：连续发送进入队列，队列项可编辑 / 重试（失败项）/ 移除；失败项以红色边框与「失败」标签提示。
- **附件**：可点击「附件」或拖拽文件随消息上传。
- **重命名会话**：弹出模态框修改标题后提交。
- **删除会话**：每个会话行可直接删除该会话（无二次确认弹框）；若删除的正是当前打开的会话，右侧面板随之清空。后端拒绝时以消息提示。
- **空态**：无会话时展示空态提示。
- **权限**：组件层面对登录用户开放全部操作，真正边界由后端按实例访问权限兜底。

#### 概览 tab（`/apps/:appId/overview`）

展示实例基础信息，并提供初始化重试、版本切换、API key 恢复与「立即重启」入口。

基础信息（描述列表）：

| 字段 | 说明 |
|---|---|
| 状态 | 实例状态徽标；初始化阶段附加进度条（拉镜像 / 起容器等无细粒度进度的阶段只显示阶段文案）；`error` 状态附加最近失败阶段与错误原因 |
| API key | 状态标签（启用 / 已禁用）；仅当状态为「已禁用」且有管理权限时显示「恢复」按钮 |
| 助手版本 | 绑定的助手版本名；`version_synced=false` 时附「需重启」标签；有权限者可「切换版本」并「立即重启」 |
| 实例语言 | 实时展示实例当前运行语言；未运行显示「实例未运行」；当前≠期望时显示需重启标签与「重启应用」按钮 |
| 所属企业 | 实例所属企业名称 |
| 描述 | 实例描述（存在时展示） |
| 镜像 ref / digest | 仅平台管理员可见（`runtime_image_ref` / `runtime_image_sha256`） |

> 概览不再展示旧版的节点、容器 ID、人设模式、模型等字段——模型与人设已并入「助手版本」，节点概念已从系统移除。

**API key 管理**：UI 仅保留「恢复」入口（禁用入口已下线，避免误关仍在使用的 key）。仅当 `api_key_status=disabled` 且有管理权限时出现「恢复」按钮，点击直接提交恢复任务，进度由 JobProgressPanel 展示。

**切换助手版本**：点击「切换版本」弹出选择框（选项取本企业允许版本 ∩ 版本目录），确认后提交；成功后提示需重启生效。可管理且可查看版本目录者可用。

**立即重启**：当 `version_synced=false` 且实例处于 `running` / `binding_waiting` 时，「助手版本」行出现「立即重启」按钮，复用 restart 任务在本页直接同步镜像，免去切到运行时 tab。

**重新初始化**：仅实例状态为 `draft` 或 `error` 且有管理权限时，右上角「重新初始化」按钮可用，点击提交初始化任务，进度由 JobProgressPanel 展示。

**需重启横幅**（顶部黄色告警，触发时出现，并带「立即重启」按钮）：

- **web 发布能力需重启**：企业已开通 web 发布但本实例尚未注入（`web_publish_pending_restart=true`）。
- **平台提示词需重启**：平台层身份 prompt 常量已更新、本实例写入的仍是旧文本（`platform_prompt_pending_restart=true`），需重启重渲染 `SOUL.md`。

**就绪 / 重启中状态**：实例状态徽标覆盖 `starting` / `restarting` / `binding_waiting` 等过渡态（黄色）；后台任务（重启 / 初始化 / 恢复 key）到达终态时，页面自动刷新状态、需重启标签与语言状态，无需手动刷新。

#### 看板 tab（`/apps/:appId/kanban`）

实例内 Hermes 任务看板，用于查看与推进助手的任务。左侧任务列表（可按关键字 / 看板筛选），右侧为任务详情、评论与执行历史；工具栏展示任务总数与最早就绪任务的等待时长。

- **创建任务**：「创建任务」按钮打开表单模态框提交。
- **任务操作**（详情面板触发）：评论 / 阻塞 / 完成 / 改派均通过 `window.prompt` 输入说明后提交；归档 / 回收通过 `window.confirm` 确认。
- **重连**：「重连」按钮直接重连实时事件流（SSE）。
- **能力探测**：写操作是否可用由后端能力位 `features.write` 决定（未知时默认展示）；若实例引擎为 stub / 不支持看板，整块替换为不支持提示。
- **权限**：查看对登录用户开放；写操作以后端能力位为准，无独立角色判断。

#### 定时任务 tab（`/apps/:appId/cron`）

管理实例内 Hermes 定时任务（Cron）。左侧任务列表（可按关键字 + 状态筛选），右侧为任务详情、执行历史与输出文件；工具栏展示网关运行状态、活跃任务数与下次执行时间。

- **创建 / 编辑任务**：通过表单模态框提交。
- **运行 / 暂停 / 恢复**：详情面板触发，直接提交，无二次确认。
- **删除任务**：`window.confirm` 确认后删除。
- **刷新**：手动刷新按钮重新拉取。
- **权限**：写操作受后端能力位 `features.write` 约束；任务名、cron 表达式（schedule）等为所有可写用户填写的常规字段（其中 schedule 必填）；一组高级字段（工作目录 workdir / 技能 skills / 模型 model / 供应商 provider / base_url）仅平台管理员且能力位 `advanced_fields` 允许时可见。stub / 不支持时整块替换为不支持提示。

#### 渠道 tab（`/apps/:appId/channels`）

绑定实例的 IM 渠道。左侧为渠道列表，当前 **4 个渠道已支持**：微信、企业微信、飞书、钉钉；另有 Telegram / WhatsApp / Discord / Slack / LINE 作为灰色「即将支持」预告，不可点击。右侧详情区随选中渠道切换。

所有渠道的发起 / 提交 / 解绑均受实例管理权限 `canManage` 约束，且要求实例「就绪」：业务状态属于 `running` / `binding_waiting` / `binding_failed` 且运行时维度 `runtime_phase=ready`。未就绪时按真实原因提示（启动中 / 重启中 / 状态确认中 / 需先恢复实例）。前端每约 4 秒轮询绑定进度。

- **微信（wechat）**：扫码绑定。点击「发起登录」由后端生成登录二维码，页面渲染二维码，用微信扫码确认；绑定成功后显示「已绑定」与绑定身份。二维码过期或处于等待 / 失败态时出现「刷新二维码」重新生成。解绑：点击「解绑」直接提交（无二次确认）。
- **企业微信（work_wechat）**：手填凭证绑定（无扫码）。填写 **Bot ID** 与 **Secret** 两个字段，两者非空且实例就绪才可「提交」；提交后清空 Secret 输入不回显。`pending_auth` 显示「验证连接中」，成功后显示已连接提示。页面内附接入指引与官方文档链接。解绑：直接提交。
- **飞书（feishu）**：扫码绑定 + 部署域选择。先选「部署域」（飞书国内 / Lark 国际，绑定后不可改），点击「发起登录」由后端异步建应用并回传二维码；扫码后显示「已扫码，正在验证并连接…」，成功后展示 bot 名称、绑定身份与生效部署域。解绑：直接提交。
- **钉钉（dingtalk）**：手填凭证绑定（无扫码）。填写 **Client ID**（即 AppKey）与 **Client Secret**（即 AppSecret），两者非空且实例就绪才可「提交」；提交后清空 Secret。`pending_auth` 显示「验证连接中」，成功后显示已连接提示，附接入指引与官方文档链接。解绑：直接提交。

失败时各渠道均在详情区以红色文案展示后端返回的错误原因（凭证错误 / 超时等）。

**权限边界**：有实例管理权限的用户（平台管理员 / 企业管理员 / 实例 owner）可操作渠道绑定与解绑。

#### 实例知识库 tab（`/apps/:appId/knowledge`）

管理该实例私有知识库文件，文件主库由 RAGFlow 承担，页面展示扁平文档列表。

- **上传**：右上角「上传文件」支持多选，也可拖拽多文件到列表卡片区域；批量上传按顺序串行执行，单文件失败不阻塞后续，进度由全局上传进度弹窗汇总。单文件上限 **1GB**，并按类型白名单过滤；上传成功即进入知识库并触发解析，不等待解析完成。
- **列表**：展示文件名、大小、类型、解析状态（含进行中进度 / 失败原因）、上传时间与操作；支持关键字搜索、按解析状态筛选与分页。顶部展示只读的容量用量汇总（已用 / 总量 / 剩余）。
- **下载**：文件行「下载」按钮直接下载。
- **删除**：文件行「删除」按钮直接删除（无二次确认弹框）。
- **重解析**：仅解析失败 / 已停止的文件显示「重解析」，点击直接重新提交解析。
- **Hermes 写入**：Hermes 内置知识库 skill 可把 `/opt/data/workspace` 下的工作产物写入当前实例知识库，只写当前实例。
- **权限**：上传 / 删除 / 重解析受实例管理权限约束；下载对可访问该实例的用户开放；平台管理员另有 RAGFlow dataset 运维入口。

> 本 tab 不再提供在线修改容量的能力（已下线），容量仅作只读展示。企业级知识库不在此处，见 `/knowledge` 页面。

#### 技能 tab（`/apps/:appId/skills`）

管理该实例安装的技能（skill）。本 tab 为壳组件，内部渲染统一的技能管理器，含「已安装 / 技能市场」等分区：

- **已安装**：列出技能名、来源（平台库 / ClawHub / 内置 / 自建）、版本与状态；有新版时可「更新到最新」（直接提交）；可「卸载」（弹框二次确认）；`pending` 状态可「重新安装」（直接提交）。当前助手版本必需的受保护技能带锁标记、不可卸载（后端 `ErrAppSkillProtected` 兜底）；内置技能只读。
- **技能市场**：按来源浏览可安装技能并「安装」，已安装项有标记。
- **写操作权限**：受 `canManageAppSkill` 约束（平台管理员 / 企业管理员本企业 / 实例 owner）；无权限时不显示操作按钮。
- 若实例引擎版本过旧不支持技能运行时，整块以提示替代。

> 技能体系（平台技能库 / 市场 / 定制工单等）的完整说明见后续技能体系章节。

#### 工作目录 tab（`/apps/:appId/workspace`）

只读浏览实例 Hermes 容器工作目录。

- **浏览**：点击目录名进入子目录；顶部面包屑显示当前相对路径，「返回上级」退回上一层；支持关键字搜索（跨整棵目录树返回完整路径）。
- **下载单文件**：文件行「下载」按钮直接下载。
- **下载归档**：右上角「下载归档」将当前目录打包 zip 下载。
- 该页面只读，manager 不支持向 workspace 写入（写入只能由 Hermes 容器进程完成）。可访问该实例的用户均可浏览与下载。

#### 审计 tab（`/apps/:appId/audit`）

展示该实例的审计记录，列包含：时间、操作者（角色徽标 + ID，系统操作仅显示「系统」，已删除用户附标记）、操作动作、明细（按 target_type + action 结构化渲染，回退到原始明细或「—」）、结果（成功 / 失败 / 部分，失败附错误信息）。

**权限边界**：受 `canViewOwnAppAudit` 约束——实例 owner（企业成员）仅可查看自有实例审计，企业管理员限本企业，平台管理员可查看全部；无权限时展示无权限提示。

> **关于 AI 人设**：企业侧不再有独立的人设编辑页。AI 人设由「助手版本」承载（平台侧维护，见 §1.4）；最终注入 Hermes 容器的 system prompt 由「平台默认 → 企业 → 实例」三层在容器内按固定顺序拼接。

### 2.5 企业级知识库

页面路径：`/knowledge`

管理企业共享知识库文件。文件主库由 RAGFlow 承担，页面展示扁平文档列表、解析状态和可用操作。上传区顶部标注可接受的文件类型（PDF、DOC、DOCX、TXT、MD、CSV、XLS、XLSX、PPT、PPTX、JPG、JPEG、PNG、GIF）与单文件上限（1024MB）。

**上传文件**：点击右上角「上传文件」可一次选择多个文件，也可以把多个文件拖拽到知识库文件列表卡片区域（拖拽上传仅在有写权限时生效）。批量上传按文件顺序串行执行；类型不在白名单的文件会被前端拦截并提示；单个文件失败不会阻塞后续文件，上传进度弹窗会汇总成功、失败和取消数量。每个文件上传完成表示该文件已进入 RAGFlow 并触发解析，不等待解析完成。

**容量用量**：列表上方展示「已用 / 上限 / 剩余」容量摘要。

**查看列表**：支持按文件名关键字和解析状态过滤；列表展示文件名、大小、类型、解析状态（解析中附进度百分比、失败附错误信息）、创建时间和可用操作，按后端总数远程分页。

**下载文件**：点击文件行的「下载」按钮，浏览器自动下载该文件。

**删除文件**：点击文件行的「删除」按钮（浏览器原生 `confirm` 弹框确认）。

**重新解析**：仅解析失败或已停止的文件显示「重解析」按钮，点击后重新提交 RAGFlow 解析任务。

**清空文件**：右上角「清空文件」将删除本企业知识库全部文件，采用输入校验值的强确认弹窗，避免误操作；知识库为空时按钮禁用。

> 不提供文件预览；需要查看原文件时请下载。
> RAGFlow 远端 dataset 运维入口（「RAGFlow 信息」按钮）仅平台管理员可见，企业管理员不显示。

**权限边界**：受 `canManageOrgKnowledge` 约束——平台管理员可跨企业浏览、下载；企业管理员可浏览、下载本企业文件并执行上传 / 删除 / 重解析 / 清空；企业成员只读浏览和下载（成员视角见 §3）。

### 2.6 企业余额

页面路径：`/balance`（仅企业管理员可见）

只读展示本企业的账户余额与充值流水，页面本身不含任何写操作（充值由平台管理员在 §1.2 的充值页发起）。

**概况卡片**（2 张）：累计充值金额（`total_recharged`）、当前剩余金额（`remain_quota`）。两者实时查询 new-api，加载中显示占位符，查询失败显示失败提示。金额按 new-api 计费配置格式化展示。

**充值记录**：表格列包含时间、金额、备注、状态（成功 / 失败），每页 15 条。企业管理员视角不展示操作人列。

**权限边界**：路由限企业管理员访问；余额与用量数据实时取自 new-api，manager 不做本地缓存。

### 2.7 web 发布（企业侧）

涉及两个页面：配置页 `/platform/web-publish-config` 与已发布站点页 `/published-sites`（均对平台管理员与企业管理员开放）。企业管理员视角锁定为本企业，无跨企业选择器。

**配置页（`/platform/web-publish-config`）**：企业管理员管理「本企业且平台已开通」的 web 发布配置。

- **可编辑**：根域名（`base_domain`，必填）、DNS provider（`dns_provider`，必填，目前仅开放阿里云 `alidns`，本地 dev 模式另含 `local`）、站点存活天数（`site_ttl_days`，默认 7）、最大站点数（`max_sites`，默认 20），以及 DNS provider 凭证（`access_key_id` / `access_key_secret`，写入即生效、write-only，保存后不回填明文，留空则保留已有密文）。
- **不可操作**：开通 / 停用 web 发布能力属基础设施层闸门，「开通 / 停用」卡片仅平台管理员可见；证书状态面板对企业管理员只读，重试签发仅平台管理员可用。

**已发布站点页（`/published-sites`）**：查看本企业已发布站点及证书状态。

- 列表列：URL（点击新标签打开）、发布者、状态（active / disabled / expired）、到期时间、大小、操作。
- **可操作**：续期（`renew`，任意状态可用）、下线（`takedown`，仅 active 站点显示，二次确认弹窗）。
- 页面下方内嵌证书状态面板，企业管理员只读，证书重试 / 续签签发仅平台管理员可用。

**权限边界**：两个页面路由均限平台管理员与企业管理员；企业侧只能操作本企业，开通 / 停用与证书重试签发保留在平台管理员手中。

### 2.8 用量

### 2.9 审计日志

展示本企业所有审计记录。企业管理员视角无跨企业切换下拉框（该下拉仅平台管理员可见）。

## 4. 通用约定

### 4.1 高风险操作确认

以下操作统一弹出 `ConfirmActionModal`，要求在弹框中输入对应对象名（实例名 / 成员登录名）后才能放行：

- 停止实例容器
- 删除实例（及容器、API key）
- 删除成员（及名下实例）
- 重置成员密码（需输入成员登录名）
- 禁用 API key

知识库文件删除使用浏览器原生 `confirm` 弹框；企业启用 / 禁用无需输入确认。

### 4.2 操作可见性

UI 隐藏的菜单和按钮仅为体验优化；后端对每一个写操作都执行独立权限校验。发现某账号能调到不该有的接口请立即上报。

### 4.3 异常状态提示

| 场景 | UI 表现 | 处置 |
|---|---|---|
| manager-api 不可用 | 页面数据无法加载 | 等运维恢复 |
| 节点 unreachable | 实例「资源指标尚未采集」 | 等心跳恢复或联系运维 |
| 实例 error | 实例状态标红 | 在「运行时」tab 查看日志，执行重启；或「概览」tab 重新初始化 |
| 渠道凭证失效 | 渠道页显示过期状态 | 解绑后重新扫码 |
| 知识库解析 failed | 知识库文件行显示解析失败和错误信息 | 点「重解析」重新提交 RAGFlow 解析任务 |
| 二维码过期 | 渠道页提示「当前二维码已过期」| 点「刷新二维码」重新生成 |

### 4.4 浏览器要求

支持最新版 Chrome / Edge / Firefox / Safari；不支持 IE。
