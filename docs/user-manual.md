# 用户手册

> 平台管理员 / 企业管理员 / 企业成员三类角色在 manager 前端能做什么。按角色分章节，每章给出页面入口、典型操作步骤与权限边界。

---

## 登录

访问 manager 部署地址（本地默认 `http://localhost:5173`），进入登录页（路由 `/login`）。

**登录表单字段**（自上而下三个输入框）：

- **企业标识**：平台管理员留空；企业用户填写所属企业的 `code`（如 `test-org`）。占位提示为「企业用户填写，平台管理员留空」。
- **账号**：登录账号。
- **密码**：登录密码；右侧「眼睛」图标可切换明文/密文显示。

页面不提供自助注册入口。若部署侧开启了人机校验，表单会常驻一枚验证码组件，加载后自动完成校验，无需手动点击。

**界面语言切换（中/英）**：登录卡片右上角有语言选择器，可在 **简体中文** 与 **English** 间切换（当前仅支持这两种语言）。默认语言优先取本机上次选择（localStorage），否则采用平台默认语言，最终兜底为英文。登录页切换会记住在本机（localStorage），跨刷新/跨会话在本机持久，但不同步到账号偏好；登录后可在顶栏语言选择器再次切换，此时才会持久化到账号偏好并跟随账号跨设备生效。

提交后系统颁发 access token + refresh token；access token 过期时前端自动用 refresh token 续签，无需手动重新登录。

**重置密码**：不提供自助找回；请联系企业管理员或平台管理员在成员管理页重置。

**登录后跳转**：进入角色首页（`RoleAwareHome`），按角色分流：

- 平台管理员（`platform_admin`）→ 平台控制台 `/console`。
- 企业管理员（`org_admin`）→ 企业控制台 `/org-console`。
- 企业成员（`org_member`）→ 其名下实例概览页；无实例时进入实例空态页 `/apps/empty`。

（若登录前因未授权被拦截，会带上 `redirect` 参数，登录成功后回跳到原目标页。）

## 1. 平台管理员

平台管理员（`platform_admin`）管理整个平台，拥有所有资源的最高访问权限。详细权限规则参见 [`docs/product-design.md`](product-design.md)。

### 1.1 平台总览

页面路径：`/console`（登录后默认落地页；旧路径 `/platform/dashboard`、`/dashboard` 均重定向至此）。

页面顶部为一条统计卡片区，下方为图表 Tab 区。

**统计卡片（六张）**：

| 指标 | 说明 |
|---|---|
| 企业数 | 已创建企业总数 |
| 成员数 | 所有企业成员总数（副标题标注「不含平台管理员」） |
| 实例数 | 所有实例总数 |
| 运行中 | 状态为 running 的实例数（绿色高亮） |
| 异常 | 状态为 error 的实例数（红色高亮） |
| 今日 Token | 今日全平台 Token 消耗，直查 new-api 实时用量；数据未就绪时显示 `—` 并标注「加载中…」或「不可用」 |

> 统计卡片已不再展示「总余额」；余额改由各企业行 / 充值页按需实时查询。

**图表 Tab（三个）**：

- **Token 趋势**：近 7 天平台 Token 消耗折线图；用量服务不可用时显示「用量服务不可用」。
- **各企业用量**：各企业 Token 用量横向柱状图；无数据时显示「暂无数据」。
- **实例状态**：实例状态占比饼图（运行中 / 停止 / 异常）；平台数据不可用时显示「平台数据不可用」。

指标与图表均为实时查询，Token 相关数据按分钟级自动刷新。

**权限边界**：仅平台管理员可访问；`/console` 是平台管理员的默认首页。

### 1.2 企业管理

页面路径：`/organizations`

**查看企业列表**，每行展示：名称（备注作为副标题）、企业标识（`code`）、状态、联系人、电话、预警阈值（`%`）、实例上限（未设置显示「不限」）、知识库空间（GB）、当前余额（对每个企业并发实时查询 new-api，未加载时显示 `…`）。

**创建企业**：

1. 点击右上角「新增企业」展开表单卡片。
2. 必填字段：
   - 名称
   - 企业标识（`code`，登录命名空间，创建后不可修改）
   - 管理员账号（`admin_username`）
   - 管理员姓名（`admin_display_name`）
   - 管理员密码（`admin_password`）
3. 可选字段：联系人、联系电话、余额预警阈值（0–100 %）、实例数量上限（留空 = 不限制）、企业知识库空间（GB）、个人知识库空间（GB，作为该企业新建实例的默认个人库上限，仅对之后新建实例生效）、备注、可用助手版本（多选，可留空）。
4. 点击「保存」提交；系统自动在 new-api 创建对应账号。

> 表单中不再设置「可用模型」，也不在此配置企业 AI 人设；企业可用能力通过「可用助手版本」多选授予。

**编辑企业**：点击对应行「编辑」，以编辑模式打开同一表单。可修改名称、联系人、电话、预警阈值、实例上限、知识库空间、个人知识库空间、备注、可用助手版本；企业标识只读展示，编辑模式不含管理员账号字段。

**启用 / 禁用企业**：在对应行点击「禁用」或「启用」（两个按钮按当前状态互斥显示）。切换直接生效，无二次确认弹框。禁用后该企业成员无法登录。

**复制企业信息**：点击「复制信息」，将企业标识、名称、管理员账号及密码提示（明文不保存）按固定格式写入剪贴板，便于转发给企业管理员；结果以页面内文字反馈。

**充值 Token Credit（列表内弹框）**：

1. 点击目标企业行的「充值」，弹出「企业充值」弹框。
2. 弹框顶部展示当前余额（剩余 / 已用，直查 new-api，不缓存）。
3. 填写充值金额（正整数）和可选备注。
4. 点击「确认充值」提交；金额非正整数时在弹框内提示，充值结果以文字反馈。

> 该弹框为直接确认，不要求输入企业名称二次校验。充值为内部记账操作，不涉及在线支付流程。

**查看充值记录**：点击对应行「充值记录」，弹出记录弹窗，顶部概况展示「累计充值金额」和「当前剩余金额」，下方表格列出时间、金额、备注、状态（成功 / 失败）、操作人。

**独立充值页**：另有独立充值页 `/platform/organizations/:orgId/recharge`（`RechargePage`），布局为充值表单 + 充值历史两张卡片。该页的确认走 `ConfirmActionModal` 二次确认，需在弹框中输入目标企业名称才能提交，历史表格额外展示错误信息列。当前 UI 未提供直达该页的入口按钮，主要通过直接访问该路由进入（页内「返回企业列表」链接回到 `/organizations`）。

**权限边界**：企业管理与充值相关页面仅平台管理员可访问。

### 1.3 权限管理

页面路径：`/platform/permissions`

**只读**的权限矩阵说明页，用于对照三类角色（平台管理员 / 企业管理员 / 企业成员）在各功能模块下的可操作范围。页面不调用任何接口、无任何可执行操作，纯静态展示。

页首有图例：`✅` 可操作（无条件）、`❌` 无权限、`🟡` 有条件（本企业 / 本企业全部 / 仅自己）。

矩阵按功能模块分组为多张表格，涵盖：企业管理、成员管理、应用实例、运行时操作、渠道（Channel）、知识库、助手版本、任务看板（Kanban）、Cron 任务、用量、审计日志、充值记录、运行时节点、平台总览、模型列表、后台任务（Jobs）、工作区、资源指标。每张表按「操作 / 平台管理员 / 企业管理员 / 企业成员」四列列出各角色权限。

> 此页仅呈现权限设计，实际鉴权以后端 `internal/auth/authorizer.go` 为准。

**权限边界**：仅平台管理员可访问。

### 1.4 审计日志（平台视角）

页面路径：`/audit-logs`

平台管理员访问此页面可跨企业查看审计日志：

1. 工具栏的企业下拉框切换目标企业。
2. 列表展示：时间、操作者角色 + ID、资源类型 + 资源 ID、操作动作、操作结果（成功 / 失败 / 部分）及错误详情。

**权限边界**：平台管理员和企业管理员均可访问此页面；企业管理员只能查看自身企业的日志。

### 1.5 用量（平台视角）

页面路径：`/usage`

平台管理员看到四个 tab：

- **平台**：全平台汇总用量。
- **企业**：企业下拉框选择目标企业后展示企业维度用量。
- **成员**：可同时选择企业和成员，查看单个成员的用量明细。
- **实例**：可同时选择企业和实例，查看单个实例的用量明细。

**权限边界**：各维度查询按角色裁剪；成员和实例维度需先选定企业。

### 1.6 成员管理（跨企业查看）

页面路径：`/members`

平台管理员通过工具栏企业下拉框切换查看各企业成员列表（只读，不可执行创建 / 禁用 / 删除操作）。

平台管理员可对已有成员点击「创建新实例」，为没有实例的成员补建一个实例：

1. 点击成员行的「创建新实例」展开表单。
2. 填写实例名、选择人设模式（沿用企业人设 / 实例覆盖）、选择模型。
3. 可选填写实例 prompt。
4. 提交后后台异步初始化容器。

### 1.7 实例列表（跨企业查看）

页面路径：`/apps`

平台管理员通过企业下拉框切换查看各企业实例列表，并可对实例执行重启 / 停止 / 删除操作（需二次确认）。

### 1.8 行业知识库

页面路径：`/platform/industry-knowledge`

平台管理员管理平台级通用行业资料，行业名称在未删除行业库中唯一。行业知识库不属于某个企业，是否被实例检索取决于助手版本配置。

**创建 / 重命名行业库**：点击「新建行业库」填写行业名称；在行业库列表中点击编辑可重命名。

**上传文件**：选择左侧行业库后，点击「上传文件」选择一个或多个文件。文件上传到该行业库对应的 RAGFlow dataset 并触发解析；同名文件会覆盖当前行业库内的旧文件。

**文件管理**：文件列表展示解析状态、大小、上传时间和操作。平台管理员可下载、删除，或对解析失败 / 已停止文件重新解析。

**删除行业库**：行业库仍被任一未删除助手版本引用时不能删除；需要先在助手版本中移除关联。

**外部上传接口文档**：点击页面右上角「接口文档」可查看外部商业知识库上传接口说明。页面会从 manager 当前配置读取 `industry_knowledge.upload_token` 并写入弹框和一键复制的 Markdown 内容，方便交付给外部服务方。

外部商业知识库同步接口也写入行业知识库。该接口使用配置项 `industry_knowledge.upload_token` 作为固定鉴权字符串，外部服务提交行业名称和文件，manager 会按行业名称找到或创建行业库。

接口地址：

```text
POST /api/v1/external/industry-knowledge/files
```

鉴权 Header：

```text
X-OC-Industry-Knowledge-Token: <industry_knowledge.upload_token>
```

表单字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| `industry_name` | 是 | 行业名称。不存在时自动创建行业库；未删除行业库中名称唯一。 |
| `file` | 是 | 上传文件。同一行业库内同名文件会覆盖旧文件。 |

curl 调用示例：

```bash
curl -i \
  -H "X-OC-Industry-Knowledge-Token: <页面显示的实际 upload_token>" \
  -F "industry_name=保险" \
  -F "file=@./policy.pdf;type=application/pdf" \
  https://<manager-domain>/api/v1/external/industry-knowledge/files
```

返回码：

| 状态码 | 说明 |
|---|---|
| `202` | 上传成功，文件已进入 RAGFlow 解析队列。 |
| `400` | 参数缺失、行业名称为空或请求体格式错误。 |
| `401` | 缺少或错误的 `X-OC-Industry-Knowledge-Token`。 |
| `413` | 文件大小超过平台上传限制。 |

上传成功后通常先返回 `parse_status=queued`，解析完成后才能稳定参与检索。外部上传只负责写入行业库；实例是否检索该行业库，由助手版本的行业知识库关联决定。

### 1.9 助手版本关联行业知识库

页面路径：`/assistant-versions`

创建或编辑助手版本时，可以在「行业知识库」字段选择一个或多个行业库。对应实例运行时执行 `oc-kb search` 时，除实例知识库和企业知识库外，还会检索这些行业库。

每个关联行业库都会单独返回最多 `top_k` 条结果。选择过多会增加上下文长度和响应成本，页面会在编辑区提示该风险，由平台管理员自行判断关联数量。

## 2. 企业管理员

企业管理员（`org_admin`）管理本企业的成员、实例、人设、知识库和用量。

### 2.1 成员管理

页面路径：`/members`

**查看成员列表**：展示本企业成员的用户名、显示名、角色、状态。

**创建成员并初始化实例**：

页面路径：`/members/new`

点击「创建并初始化」进入专用表单，一步完成成员账号和配套实例的创建：

**账号信息**：

| 字段 | 说明 |
|---|---|
| 用户名 | 同一企业下唯一 |
| 显示名 | UI 展示用 |
| 初始密码 | 至少 8 位 |
| 角色 | 企业成员 / 企业管理员 |

**实例信息**：

| 字段 | 说明 |
|---|---|
| 实例名 | 必填 |
| 人设模式 | 沿用企业人设 / 实例覆盖 |
| 模型 | 从企业可用模型列表中选择 |
| 实例 prompt | 可选，仅人设模式为「实例覆盖」时生效 |

提交后后台 worker 异步初始化容器，表单区域展示创建结果（成员 + 实例 + Job ID）。

**仅创建账号（不初始化实例）**：

回到 `/members` 列表，点击「新增成员」展开表单，填写用户名、显示名、初始密码和角色。此方式不创建实例。

**禁用 / 启用成员**：

在成员行点击「禁用」或「启用」；禁用后该成员无法登录，实例容器维持原状态。

**重置密码**：

1. 点击「重置密码」，弹出提示框（`window.prompt`）输入新密码（至少 8 位）。
2. 输入有效后弹出「确认重置成员密码」弹框，需输入成员登录名确认。
3. 点击「确认重置」，原密码立即失效。

**删除成员**：

1. 点击「删除」，弹出「确认删除成员」弹框。
2. 弹框说明：将禁用账号并提交名下实例的删除任务，操作不可撤销。
3. 点击「确认删除」执行；后台 worker 停止并删除实例容器、注销 API key。

**权限边界**：企业管理员只能管理本企业成员；平台管理员在此页面为只读查看（可额外执行「创建新实例」操作）。当前登录用户无法删除自身账号。

### 2.2 实例列表与快速操作

页面路径：`/apps`

展示本企业所有实例，包含实例名（链接跳转详情）、状态、API key 状态、容器 ID。

工具栏提供「创建成员并初始化」快捷入口（跳转至 `/members/new`）。

列表操作列可直接对实例执行：

- **重启**：点击后立即提交重启任务。
- **停止**：弹出「确认停止容器」弹框，需输入实例名确认。
- **删除**：弹出「确认删除实例」弹框，需输入实例名确认。

**权限边界**：企业管理员可管理本企业所有实例；企业成员只能看到自己的实例，管理权限同样适用（参见下方 §2.3）。

### 2.3 实例详情

页面路径：`/apps/:appId`（默认跳转至 `/apps/:appId/overview`）

实例详情页包含六个 tab：概览、运行时、渠道、实例知识库、工作目录、审计。

#### 概览 tab（`/apps/:appId/overview`）

展示实例基础信息：状态、API key 状态（启用 / 已禁用）、容器 ID、Runtime Node、人设模式、模型、所属企业、描述。

**API key 管理**：

- 点击「禁用」→ 弹出「确认禁用 API key」弹框 → 确认后提交禁用任务。禁用后 Hermes 容器无法调用模型。
- 点击「恢复」直接提交恢复任务。

**切换模型**：

1. 在概览页底部选择新模型。
2. 点击「保存并重启实例」（容器已创建时）或「保存模型」。
3. 提交后弹出 JobProgressPanel 展示任务进度。

**重新初始化**（仅实例状态为 `draft` 或 `error` 时可用）：

点击右上角「重新初始化」提交初始化任务，JobProgressPanel 显示执行进度。

#### 运行时 tab（`/apps/:appId/runtime`）

展示容器当前状态、容器 ID 和镜像信息，以及实例 CPU / 内存 / 磁盘读写 / 网络 RX/TX 的资源趋势图（时间范围：1h / 24h / 7d / 30d）。

右上角操作按钮：

| 操作 | 可用条件 | 确认方式 |
|---|---|---|
| 启动 | 实例状态为 `stopped` | 直接提交 |
| 停止 | 实例状态为 `running` 或 `binding_waiting` | 弹框确认，需输入实例名 |
| 重启 | 同停止 | 直接提交 |
| 删除 | 实例未处于 `deleted` 状态 | 弹框确认，需输入实例名 |

操作提交后显示 Job ID 并通过 JobProgressPanel 轮询执行进度。

#### 渠道 tab（`/apps/:appId/channels`）

当前支持微信渠道绑定。

**发起登录（绑定微信）**：

1. 点击「发起登录」按钮，后端触发 Hermes 容器内微信插件生成登录二维码。
2. 页面渲染二维码图片（QR Code）；用微信 App 扫码并在手机端确认。
3. 后端轮询插件状态直至绑定成功，页面显示「已绑定」及微信昵称。
4. 二维码过期（`expires_at` 字段）后，点击「刷新二维码」重新生成。

**解绑**：

点击「解绑」按钮解绑当前微信账号，需后端接口成功返回后生效。

**权限边界**：有实例管理权限的用户（企业管理员 + 实例 owner）可操作渠道绑定。

#### 实例知识库 tab（`/apps/:appId/knowledge`）

管理该实例的私有知识库文件。文件主库由 RAGFlow 承担，页面展示扁平文档列表。

**上传**：

右上角「上传文件」按钮（`<label>` 包装原生 `file input`）支持一次选择多个文件，也可以把多个文件拖拽到知识库文件列表卡片区域。批量上传会按文件顺序串行执行，每个文件上传后立即进入当前实例知识库并触发解析；单个文件失败不会阻塞后续文件，上传进度弹窗会汇总成功、失败和取消数量。单文件上限为 1GB。上传成功不等待解析完成，解析状态后续异步刷新。

**查看列表**：

列表展示文件名、大小、类型、解析状态、解析进度 / 错误、上传时间和可用操作。

**下载文件**：

点击文件行的「下载」按钮，浏览器自动下载该文件。

**删除**：

点击文件行的「删除」按钮直接删除该文件（无二次确认弹框）。

**重新解析**：

仅解析失败或已停止的文件显示「重解析」按钮。点击后重新提交 RAGFlow 解析任务。

**Hermes 写入**：

Hermes 内置知识库 skill 可以把 `/opt/data/workspace` 下的工作产物加入当前实例知识库。
该能力只写当前实例，不会写企业知识库或其他实例知识库。

**权限边界**：有实例读取权限的用户可浏览和下载；有实例管理权限的用户可上传、删除和重新解析。企业级知识库在此 tab 不可见（企业级知识库在独立的 `/knowledge` 页面管理）。

#### 工作目录 tab（`/apps/:appId/workspace`）

浏览 Hermes 容器工作目录（只读）。

**浏览**：点击目录名进入子目录；顶部面包屑显示当前相对路径，点「返回上级」退回上一层。

**下载单个文件**：点击文件行的「下载」按钮，浏览器自动下载。

**下载归档**：右上角「下载归档」按钮将当前目录打包为 zip 下载。

> manager 不允许通过此页面向 workspace 写入文件，写入只能由 Hermes 容器进程完成。

**权限边界**：所有有权访问该实例的用户均可浏览和下载工作目录文件。

#### 审计 tab（`/apps/:appId/audit`）

展示该实例的审计记录，列包含时间、操作者角色 + ID、操作动作、操作结果（成功 / 失败）及错误详情。

**权限边界**：实例 owner（企业成员）只能查看自己实例的审计；企业管理员和平台管理员也可查看。

### 2.4 企业 AI 人设

页面路径：`/org/persona`

设置企业默认 AI 人设，支持以下字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| 系统提示词 | 是 | 注入 Hermes 容器的核心人设内容 |
| 对话规范 | 否 | 补充对话行为约束 |
| 禁止事项 | 否 | 明确列出禁止内容 |
| 回复风格 | 否 | 期望的回复风格描述 |
| 允许成员实例覆盖 | — | 勾选后成员可通过实例 prompt 覆盖企业默认人设 |

点击「保存新版本」提交；系统创建新版本（旧版本保留），下次容器重建时生效。页面显示当前生效版本号。

最终注入容器的 system prompt 由「平台默认 → 企业 → 实例」三层拼接，顺序固定。

**权限边界**：平台管理员和企业管理员可编辑；企业成员只读（表单不可提交）。

### 2.5 企业级知识库

页面路径：`/knowledge`

管理企业共享知识库文件。文件主库由 RAGFlow 承担，页面展示扁平文档列表、解析状态和可用的重解析操作。

**上传文件**：点击右上角「上传文件」可一次选择多个文件，也可以把多个文件拖拽到知识库文件列表卡片区域。批量上传按文件顺序串行执行；单个文件失败不会阻塞后续文件，上传进度弹窗会汇总成功、失败和取消数量。单文件上限为 1GB。每个文件上传完成表示该文件已进入 RAGFlow 并触发解析，不等待解析完成。

**查看列表**：列表展示文件名、大小、类型、解析状态、解析进度 / 错误、上传时间和可用操作。

**下载文件**：点击文件行的「下载」按钮，浏览器自动下载该文件。

**删除文件**：点击文件行的「删除」按钮（浏览器原生 `confirm` 弹框确认）。

**重新解析**：仅解析失败或已停止的文件显示「重解析」按钮。点击后重新提交 RAGFlow 解析任务。

> 第一版不提供文件预览；需要查看原文件时请下载。

**权限边界**：平台管理员可跨企业浏览和下载；企业管理员和企业成员可浏览、下载本企业文件；上传 / 删除操作仅限本企业管理员；企业成员只读浏览和下载。

### 2.6 用量

页面路径：`/usage`

企业管理员看到三个 tab：

- **企业**：本企业汇总用量，直查 new-api 实时数据。
- **成员**：选择成员后查看其用量明细。
- **实例**：选择实例后查看其用量明细（实例需已绑定 new-api key）。

**权限边界**：企业管理员可查看本企业所有维度；企业成员只能看「我的用量」tab。

### 2.7 审计日志

页面路径：`/audit-logs`

展示本企业所有审计记录，字段与平台视角相同（无跨企业切换下拉框）。

**权限边界**：仅企业管理员和平台管理员可访问。

## 3. 企业成员

企业成员（`org_member`）管理自己名下唯一绑定的 Hermes 实例。登录后自动跳转至实例列表（只展示自己的实例）。

### 3.1 实例列表

页面路径：`/apps`

只展示属于自己的实例。可点击实例名进入详情，或在操作列直接执行重启 / 停止 / 删除（需二次确认）。

### 3.2 实例详情

页面路径：`/apps/:appId`

与企业管理员操作路径相同（参见 §2.3），权限范围限定为自身实例：

- **概览 tab**：查看基础信息；可切换模型、禁用 / 恢复 API key、在 `draft`/`error` 状态下重新初始化。
- **运行时 tab**：查看资源趋势；启动 / 停止 / 重启 / 删除容器。
- **渠道 tab**：发起微信登录绑定 / 解绑。
- **实例知识库 tab**：上传 / 浏览 / 下载 / 删除实例私有知识库文件。
- **工作目录 tab**：浏览工作目录；下载单个文件；下载当前目录归档。
- **审计 tab**：查看自身实例的审计记录。

### 3.3 用量

页面路径：`/usage`

企业成员只能看到「我的用量」tab，展示自己实例的用量明细。

### 3.4 企业 AI 人设（只读）

页面路径：`/org/persona`

企业成员可浏览当前生效的企业人设版本，但无法编辑或保存。

若企业管理员开启了「允许成员实例覆盖」，则在创建成员时（由管理员操作）或创建新实例时可以指定实例 prompt 来覆盖企业默认人设。

### 3.5 知识库（只读浏览）

页面路径：`/knowledge`

企业成员可浏览并下载企业共享知识库文件列表，但无法上传、删除或重新解析文件。

## 4. 通用约定

### 4.1 高风险操作确认

以下操作统一弹出 `ConfirmActionModal`，要求在弹框中输入对应对象名（实例名 / 成员登录名）后才能放行：

- 停止实例容器
- 删除实例（及容器、API key）
- 删除成员（及名下实例）
- 重置成员密码（需输入成员登录名）
- 禁用 API key

知识库文件删除使用浏览器原生 `confirm` 弹框；企业启用 / 禁用无需输入确认。

### 4.2 操作可见性

UI 隐藏的菜单和按钮仅为体验优化；后端对每一个写操作都执行独立权限校验。发现某账号能调到不该有的接口请立即上报。

### 4.3 异常状态提示

| 场景 | UI 表现 | 处置 |
|---|---|---|
| manager-api 不可用 | 页面数据无法加载 | 等运维恢复 |
| 节点 unreachable | 实例「资源指标尚未采集」 | 等心跳恢复或联系运维 |
| 实例 error | 实例状态标红 | 在「运行时」tab 查看日志，执行重启；或「概览」tab 重新初始化 |
| 渠道凭证失效 | 渠道页显示过期状态 | 解绑后重新扫码 |
| 知识库解析 failed | 知识库文件行显示解析失败和错误信息 | 点「重解析」重新提交 RAGFlow 解析任务 |
| 二维码过期 | 渠道页提示「当前二维码已过期」| 点「刷新二维码」重新生成 |

### 4.4 浏览器要求

支持最新版 Chrome / Edge / Firefox / Safari；不支持 IE。
