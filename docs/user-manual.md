# OpenClaw Manager 用户手册

面向终端使用者的操作指南。涵盖三类角色：

- **平台管理员（platform admin）** — 管理整个平台、查看和启停节点、开通组织、给组织充值
- **组织管理员（org admin）** — 管理本组织成员、应用、人设、知识库
- **组织成员（org member）** — 管理自己名下唯一的 OpenClaw 应用

部署、运维相关请看 [`deploy/README.md`](../deploy/README.md)。

## 1. 登录

打开浏览器访问 manager 部署地址（本地默认 `http://localhost:5173`，生产域名见运维公告）。

| 角色 | 账号来源 |
|---|---|
| 平台管理员 | 通过 `cmd/seed-admin` 注入或由其他平台管理员创建 |
| 组织管理员 / 组织成员 | 由组织管理员在「成员管理」中创建 |

组织用户登录需要填写组织标识、用户名和密码。平台管理员登录时组织标识留空。
组织名称是展示名称，可以由平台管理员修改；组织标识是登录命名空间，创建后不修改。

登录后系统按角色显示不同左侧菜单。Cookie 同时颁发 access token + refresh token，
access token 默认 15 分钟有效，过期时前端用 refresh token 自动续签。

> **重置密码**：当前版本不提供 self-service 重置；请联系组织管理员或平台管理员
> 在「成员管理」中重置。

## 2. 平台管理员

### 2.1 平台总览

进入 **平台总览** 页查看全平台六项核心计数：组织数、成员数、应用数、运行容器数、
活跃节点数、过去 24h 异常 job 数。任一项出现红字告警时按提示跳转排查。

### 2.2 组织管理

**组织管理** 页可：

- 创建组织：填写组织名、联系人、联系方式，提交后系统自动在 `new-api` 创建对应账号；
- 启用 / 禁用组织：禁用后该组织成员无法登录，应用容器维持原状态但不再消耗 token；
- 查看组织余额（直查 `new-api`，不缓存）；
- 充值 Token Credit：在组织行的「充值」入口填写金额，系统会调 `new-api` 累加该组织
  账号余额并写审计。

> 第一版充值是「内部记账动作」，不涉及人民币 / 发票 / 在线支付。

### 2.3 注册 Runtime Node

第一次部署需要至少一个节点；否则任何应用创建都会失败（节点容量不足）。

1. **生成 enrollment secret**：使用 `openssl rand -base64 32`，写入
   `manager.yaml` 的 `runtime.enrollment_secret`，并写入每台节点 `agent.yaml`
   的 `manager.enrollment_secret`。
2. **运维准备节点**：在节点上启动 `oc-runtime-agent` 容器，配置：
   - `agent.name`：节点展示名
   - `agent.advertise_host`：manager 能访问到的节点主机名或 IP
   - `agent.max_apps`：节点最大可承载应用数；留空表示不限
   - `agent.data_root` / `agent.state_dir`
   - `manager.endpoint`
   - `manager.enrollment_secret`
3. **agent 自动注册**：agent 首次启动会调 manager `POST /api/v1/agent/enroll`，
   自动创建或刷新 runtime node，并把 `node_id` / `agent_token` 写入 `state_dir`。
4. **后台查看**：打开 **运行节点** 页，新节点会自动出现。后台不再提供「注册节点」
   或「Rotate bootstrap token」操作。
5. **token 异常恢复**：如果 agent 心跳收到 401，会自动重新 enroll；如需强制刷新，
   重启对应 agent 即可。

详细跨机部署步骤见 [`deploy/README.md`](../deploy/README.md) 「跨机部署（多 runtime node）」。

### 2.4 节点容量与健康

**运行节点** 列表展示每个节点的：

- 状态（待注册 / 活跃 / 不可达 / 禁用）
- 探测状态与最近探测错误；`degraded` 表示 manager 到 agent 入站端口异常
- 最近心跳时间
- 当前应用数 / 最大应用数
- agent 版本、Docker 版本、OS 内核
- 资源摘要（CPU / 内存 / 磁盘）

`unreachable` 持续超过运维约定阈值（默认 90s）时，受影响应用会显示「资源指标尚未采集」。
心跳恢复后自动回 `active`，无需手动干预。

### 2.5 全平台审计

**审计日志** 页支持按 操作人 / 组织 / 动作 / 时间范围 过滤。鼠标悬停 `target_id`
查看完整 ID。审计写入分散在各业务 service，不会因 worker 故障漏写。

## 3. 组织管理员

登录后进入本组织 dashboard，看到组织余额、成员数、应用数、本月 token 用量四项摘要。

### 3.1 创建成员（含初始化应用）

**重点**：第一版「成员账号 = 应用 owner」一对一强绑定。组织管理员**必须**在创建账号
的同一表单里完成应用初始化，没有独立的「应用创建向导」。

进入 **成员管理 → 创建成员**，填写：

| 字段 | 说明 |
|---|---|
| 登录账号 | 全局唯一 |
| 显示名称 | UI 展示用 |
| 角色 | 组织成员 / 组织管理员（组织管理员也会创建一个属于他自己的应用） |
| 初始密码 | 系统强校验：≥ 12 位、大小写 / 数字 / 符号至少各 1 |
| 应用名 | 同一组织下唯一 |
| 应用描述 | 可选 |

> 第一版「节点选择」字段已下线：服务端 `node_selector` 自动选 **剩余容量最大** 的
> 活跃节点。若要调整容量，请修改对应 runtime-agent 的 `agent.max_apps` 并重启 agent，
> manager 会在重新 enroll 后同步新值。

提交后：

1. 立即返回成员 + 应用对象；
2. 后台 `member_onboarding` job 启动：分发镜像、创建容器、申请 `new-api api_key`、
   注入 system prompt 与 OpenAI 兼容配置、启动容器；
3. 应用列表项显示「初始化中」直到 job 完成；失败会标红并展示错误。

### 3.2 成员账号生命周期

**成员管理** 列表可：

- 重置密码：弹窗输入新密码（同样 ≥ 12 位强校验）；
- 禁用 / 启用：禁用后该成员无法登录，但其应用容器维持原状态；
- 删除成员：**双重确认**，需要在 modal 中输入成员显示名才放行；触发删除时
  worker 会停止并删除该成员名下应用容器、注销 `new-api api_key`、清理工作目录归档。

### 3.3 组织 AI 人设（persona）

**组织 → 人设** 页设置：

- 组织级 AI 人设内容（多行文本）；
- 是否允许成员覆盖应用级人设（开关）。

最终注入 OpenClaw 容器的 system prompt 由「平台默认 → 组织 → 应用」三层拼接，顺序固定。

### 3.4 组织级知识库

进入 **知识库** 页，按目录树组织文件：

- 上传：拖拽或点击「上传文件」，支持多文件；类型 / 大小限制由后端校验；
- 删除：单文件 / 整个文件夹删除（双重确认）；
- 同步状态：每行显示该文件在组织下应用所在每个节点的同步状态
  （`pending` / `syncing` / `synced` / `failed`），`failed` 可点击「重试」入队。

> 操作完成后**异步**推送到节点，UI 上的同步状态会自动刷新。容器创建时
> manager 会先把整个组织+应用知识库一次性 tar 流到节点，再启动容器。

### 3.5 组织 token 用量

**用量** 页展示本组织本月 / 自定义日期范围的 token 用量，按成员 / 应用聚合。
所有数字直查 `new-api`，无缓存。

## 4. 组织成员

每个成员账号绑定唯一一个 OpenClaw 应用。登录后默认跳到自己的应用详情。

### 4.1 应用详情

应用详情页五个 tab：

#### 概览（Overview）

- 应用名、描述、所在节点、容器 ID、创建时间
- 当前状态（pending / creating / running / stopped / error）
- 最近一次状态变更时间
- 关联的 `new-api api_key`（仅展示后 4 位）
- 自动重启策略（运维默认开启）

#### 运行（Runtime）

控制容器生命周期：

| 操作 | 说明 |
|---|---|
| 启动 | 容器 `created` 或 `stopped` 状态时可用 |
| 停止 | 优雅停止（SIGTERM 后超时 SIGKILL） |
| 重启 | stop + start |
| 查看日志 | 滚动尾部 stdout/stderr，可选时间窗 |
| 资源指标 | CPU / 内存 / 磁盘最近一次采样（agent 上报） |
| 健康探针 | OpenClaw `/healthz`，最近探测时间 + 结果 |

> 高风险操作（停止 / 重启 / 删除）会弹出 `ConfirmActionModal`，要求输入应用名才能放行。

#### 渠道（Channels）

第一版仅微信扫码：

1. 点 **绑定微信** 弹出 modal，后端调起 OpenClaw 容器内的微信插件；
2. 容器返回二维码 payload，UI 用 `qrcode` 渲染成图片；
3. 微信 App 扫码 + 确认；
4. 后端轮询插件直至成功，应用状态更新为「已绑定」并显示微信昵称；
5. 失败 / 超时可点 **重试**；
6. **解绑** 双重确认后清理凭证。

> 二维码 payload 30 秒内有效，超时后需重新触发绑定。

#### 知识库（Knowledge）

应用级私有知识库：上传 / 删除 / 浏览，与组织级流程一致；同步只针对本应用所在节点。
对组织级知识库**只读可见**，不能编辑。

#### 工作目录（Workspace）

OpenClaw 容器内 `/workspace` 的浏览入口（只读）：

- 面包屑导航；
- 单文件下载；
- 文件夹打包下载（zip）；
- 自动归档保留期由部署侧 `openclaw.workspace.archive_retention_days` 决定。

> manager **不允许**用户向 workspace 写入文件，写入只能由 OpenClaw 容器进程完成。

### 4.2 应用级 AI 人设

仅当组织管理员勾选了「允许成员覆盖」时，应用详情页才出现 **AI 人设** 入口。否则
应用 system prompt 完全继承组织人设，组织人设变更后实时生效（容器下次冷启动时重新注入）。

### 4.3 token 用量

**用量** 入口展示自己应用本月 / 自定义日期范围的 token 用量明细。

## 5. 通用约定

### 5.1 高风险操作

平台管理员、组织管理员、组织成员的若干高风险操作统一使用 `ConfirmActionModal`：

- 删除组织
- 禁用 / 删除成员
- 停止 / 重启 / 删除应用容器
- 解绑渠道
- 删除知识库文件 / 文件夹

弹窗中需输入对应对象名（如成员显示名 / 应用名 / 文件名）才能放行，避免误操作。

### 5.2 操作可见性

UI 上隐藏的菜单和按钮**仅是体验优化**。后端会对每一个写操作做权限校验，不要假定隐藏 = 安全。
若发现某用户能调到不该有的接口，立刻按 release 流程上报。

### 5.3 异常态提示

| 场景 | UI 表现 | 处置 |
|---|---|---|
| manager-api 不可用 | 整个 dashboard 报「网络异常」 | 等运维恢复 |
| 节点 unreachable | 应用「资源指标尚未采集」 | 等心跳恢复或联系运维 |
| 容器 error | 应用 tag 红色 + 错误简要 | 看「运行 → 日志」定位；运行 → 重启 |
| 渠道 stale | 微信「凭证失效」 | 解绑后重新扫码 |
| 知识库同步 failed | 文件行红色感叹号 | 点「重试」入队 `knowledge_sync_node` |

### 5.4 浏览器要求

支持最新版 Chrome / Edge / Firefox / Safari；不再支持 IE。

## 6. 附：常见任务速查

| 想做什么 | 入口 |
|---|---|
| 加一个新组织 | 平台管理员 → 组织管理 → 创建组织 |
| 给组织充 Token Credit | 平台管理员 → 组织管理 → 行内 充值 |
| 加一个新成员（含应用） | 组织管理员 → 成员管理 → 创建成员 |
| 重置成员密码 | 组织管理员 → 成员管理 → 行内 重置密码 |
| 注册新 Runtime Node | 平台管理员 → 运行节点 → 注册节点 |
| 上传组织级知识库 | 组织管理员 → 知识库 |
| 上传应用知识库 | 成员/组织管理员 → 应用 → 知识库 tab |
| 启停应用容器 | 成员/组织管理员/平台管理员 → 应用 → 运行 tab |
| 看应用日志 | 应用 → 运行 tab → 查看日志 |
| 绑定微信 | 应用 → 渠道 tab → 绑定微信（扫码） |
| 下载工作目录文件 | 应用 → 工作目录 tab |
| 看用量 | 顶部 用量 入口（按角色聚合不同维度） |
| 看审计 | 平台/组织管理员 → 审计日志 |
