# OpenClaw Manager 产品需求与设计文档

日期：2026-04-28

## 1. 背景与目标

OpenClaw Manager 是一个面向组织的在线管理后台。用户可以在页面中创建 OpenClaw 应用，将应用绑定到微信，并通过 `new-api` 按 token 或点数计费。

系统由三个核心外部组件和一个自研后台组成：

- `ollama`：本地运行 AI 模型。
- `new-api`：模型网关，负责模型配置、账号余额、api_key、token 计费、用量日志和统计。
- `OpenClaw`：每个应用独立运行的 AI 应用实例，负责微信插件、对话能力和知识库处理。
- `manager`：本产品要实现的管理后台，负责组织、成员、应用、容器、微信绑定、知识库入口、预算策略、审计和统计展示。

产品目标：

- 平台管理员可以开通组织账号，并给组织充值 Token Credit。
- 组织管理员可以管理组织成员、预算策略、组织 AI 人设和组织下所有应用。
- 组织成员可以通过向导创建自己的 OpenClaw 应用，并绑定微信账号。
- 每个 OpenClaw 应用对应一个 Docker 容器、一个 `new-api api_key` 和最多一个微信绑定。
- manager 展示应用、成员、组织和平台维度的 token 用量，但不承担计费事实来源。
- manager 支持容器启停、日志、资源监控、健康检查和自动重启策略。

## 2. 明确边界

manager 负责：

- 平台管理员、组织管理员、组织成员三类账号和登录。
- 组织生命周期管理。
- 组织充值 Token Credit 的后台操作记录和 `new-api` 调用编排。
- 组织成员管理。
- 组织级 AI 人设和成员是否允许覆盖的策略。
- OpenClaw 应用创建向导。
- 每个应用对应的 Docker 容器生命周期管理。
- 每个应用对应的 `new-api api_key` 创建、禁用、恢复和映射。
- 微信扫码绑定流程的触发、状态展示和重试。
- 应用私有知识库文件上传入口、权限校验、状态展示和删除转发。
- 成员预算、预算预警和超额处理策略。
- token 用量查询和展示。
- 审计日志。

manager 不负责：

- Ollama 模型安装、下载、删除、参数配置或运行状态管理。
- 大模型配置、模型密钥和模型路由，这些统一由 `new-api` 管理。
- token 价格计算、扣费、真实支付、人民币金额、发票或在线支付。
- 知识库 OCR、文本切分、embedding、向量库写入或检索。
- OpenClaw 流量代理、外部访问域名分配或统一反向代理。
- 多节点调度系统、复杂日志检索和告警平台。

## 3. 部署形态

第一版采用单机一体化部署：

- `manager`
- `new-api`
- `ollama`
- 多个 OpenClaw Docker 容器

都可以部署在同一台服务器上。

设计上预留 `Runtime Node` 概念：

- 第一版只有 `local` 节点，manager 直接管理本机 Docker。
- 应用记录中保留运行节点字段。
- 后续可以扩展为多运行节点，但当前需求不包含调度、迁移、跨节点高可用。

## 4. 用户角色与权限

系统固定三类角色，不做完整 RBAC。

### 4.1 平台管理员

能力：

- 登录平台后台。
- 创建、启用、禁用组织。
- 为组织充值 Token Credit。
- 查看所有组织、成员、应用、容器状态和 token 用量。
- 查看全平台审计日志。
- 管理平台管理员账号。

限制：

- 不直接修改组织成员的应用内容，除非通过平台管理入口执行运维或风控操作。

### 4.2 组织管理员

能力：

- 登录本组织后台。
- 创建、禁用、重置组织成员账号。
- 查看和管理本组织所有 OpenClaw 应用。
- 设置组织级 AI 人设。
- 设置是否允许成员覆盖应用级 AI 人设。
- 配置成员预算、预算阈值和超额处理策略。
- 查看本组织 token 用量、成员用量和应用用量。
- 查看本组织审计日志。

限制：

- 不能访问其他组织数据。
- 不能给组织充值，只能查看组织余额和消耗。

### 4.3 组织成员

能力：

- 登录组织后台。
- 创建和管理自己名下的 OpenClaw 应用。
- 在权限策略允许时编辑应用级 AI 人设。
- 上传和管理自己应用的知识库文件。
- 绑定和重试绑定微信账号。
- 启动、停止、重启自己的应用容器。
- 查看自己的应用状态、日志和 token 用量。

限制：

- 不能查看其他成员应用，除非其角色是组织管理员。
- 不能调整成员预算。
- 不能访问组织级配置。

权限校验必须由后端 API 执行。前端菜单和按钮隐藏只作为体验优化。

## 5. 核心对象模型

### 5.1 Organization

组织代表一个租户、客户、学校或机构。

字段建议：

- 组织 ID
- 组织名称
- 状态：启用、禁用
- 联系人
- 联系方式
- 备注
- `new-api` 账号 ID
- Token Credit 余额展示字段
- 创建时间
- 更新时间

关系：

- 一个组织对应 `new-api` 中的一个账号。
- 一个组织拥有多个成员。
- 一个组织拥有多个 OpenClaw 应用。

### 5.2 Member

成员是组织内的后台账号。

字段建议：

- 成员 ID
- 组织 ID
- 登录账号
- 密码哈希
- 显示名称
- 角色：组织管理员、组织成员
- 状态：启用、禁用
- 成员预算额度
- 预算预警阈值
- 最近登录时间
- 创建时间
- 更新时间

成员不映射 `new-api` 账号。成员预算由 manager 本地维护。

### 5.3 OpenClaw App

OpenClaw 应用是产品核心运营单元。

字段建议：

- 应用 ID
- 组织 ID
- 成员 ID
- 应用名称
- 描述
- 应用状态
- AI 人设来源：组织继承、应用覆盖
- 应用级 AI 人设内容
- Runtime Node ID
- Docker 容器 ID
- `new-api api_key` ID
- api_key 状态
- 微信绑定状态
- 创建时间
- 更新时间
- 删除时间

每个应用对应：

- 一个 Docker 容器。
- 一个 `new-api api_key`。
- 最多一个微信账号绑定。
- 一份应用私有知识库。

### 5.4 Runtime Node

运行节点抽象。

第一版字段建议：

- 节点 ID
- 节点名称，例如 `local`
- Docker endpoint
- 状态
- CPU、内存、磁盘资源摘要
- 创建时间
- 更新时间

### 5.5 WeChat Binding

微信绑定记录。

字段建议：

- 绑定 ID
- 应用 ID
- 微信账号标识
- 渠道名称
- 状态：未绑定、等待扫码、已绑定、绑定失败、已解绑、凭证失效
- 二维码过期时间
- 最近在线时间
- 绑定时间
- 错误信息

每个应用最多一个微信绑定。

### 5.6 Knowledge File

知识库文件属于应用。

字段建议：

- 文件 ID
- 应用 ID
- 文件名
- 文件类型
- 文件大小
- 上传人
- 上传时间
- OpenClaw 返回的文件引用 ID
- 处理状态
- 错误信息

manager 不解析文件内容。

### 5.7 Audit Log

审计日志记录敏感操作。

字段建议：

- 日志 ID
- 操作人 ID
- 操作人角色
- 组织 ID
- 目标类型
- 目标 ID
- 动作
- 结果：成功、失败
- 错误信息
- 来源 IP
- 创建时间

## 7. 应用创建向导

OpenClaw 应用创建采用向导式流程。流程允许中途退出，并保留当前状态。

### 7.1 步骤一：基础信息

用户填写：

- 应用名称
- 描述
- 所属成员

普通成员只能给自己创建应用。组织管理员可以为组织内任意成员创建应用。

### 7.2 步骤二：AI 人设

默认继承组织级 AI 人设。

组织管理员可以配置：

- 组织级系统提示词。
- 对话规则。
- 禁止行为。
- 默认回复风格。
- 是否允许成员覆盖应用级人设。

如果组织允许覆盖，成员可以编辑应用级人设；否则只读展示组织人设。

### 7.3 步骤三：知识库上传

该步骤可跳过。

支持文件类型：

- PDF
- Word
- 图片
- 试卷文件

manager 校验：

- 文件类型
- 文件大小
- 上传人权限
- 目标应用归属

上传后直接交给 OpenClaw。上传失败不阻塞应用创建，但必须记录失败状态和错误信息。创建后仍可在应用详情继续上传或删除。

### 7.4 步骤四：初始化运行资源

进入微信绑定前，manager 执行：

1. 在组织对应 `new-api` 账号下创建应用专用 api_key。
2. 创建应用对应 Docker 容器。
3. 将 api_key、应用 ID、组织 ID、OpenClaw 配置、人设配置、知识库目录等写入容器环境变量或配置文件。
4. 启动容器。
5. 检查容器状态和 OpenClaw 健康状态。

### 7.5 步骤五：微信绑定

manager 在容器内触发 OpenClaw 微信插件流程：

- 安装或启用 `openclaw-weixin` 插件。
- 执行 `openclaw channels login --channel openclaw-weixin`。
- 获取二维码并展示给用户。
- 轮询绑定状态。

用户扫码成功后，manager 记录微信账号标识、绑定时间和状态。

如果绑定失败：

- 应用保留在待绑定或绑定失败状态。
- 容器继续运行。
- api_key 保留。
- 用户可以重试绑定。

### 7.6 步骤六：确认发布

微信绑定成功后，用户确认发布。

发布后应用进入运行中状态。

## 8. 应用状态

应用状态建议：

- `draft`：基础信息已保存，尚未创建容器和 api_key。
- `initializing`：正在创建 api_key、容器或写入配置。
- `binding_waiting`：容器已启动，等待微信扫码。
- `binding_failed`：二维码过期、扫码失败、插件错误或登录命令失败。
- `ready`：微信已绑定，等待用户确认发布。
- `running`：已发布且容器运行中。
- `stopped`：应用存在但容器停止。
- `error`：容器异常、配置错误或 OpenClaw 健康检查失败。
- `budget_limited`：容器运行，但 api_key 因预算超额被禁用。
- `deleted`：应用软删除，容器已删除，api_key 已禁用。

## 9. 创建与删除的补偿策略

资源创建顺序：

1. 保存应用草稿。
2. 创建 `new-api api_key`。
3. 创建 Docker 容器。
4. 启动容器。
5. 触发微信绑定。
6. 发布应用。

失败处理：

- api_key 创建失败：应用停留在草稿或初始化失败。
- 容器创建失败：禁用或清理已创建的 api_key，并记录错误。
- 容器启动失败：应用进入错误状态，保留 api_key，允许重试。
- 微信绑定失败：保留容器和 api_key，允许重试。
- 用户中途退出：保留当前状态，用户下次继续。

删除应用：

- 采用软删除。
- 停止并删除 Docker 容器。
- 禁用 `new-api api_key`。
- 保留应用记录、用量映射、审计记录和充值/预算相关历史。

## 10. 计费、余额与预算

### 10.1 计费事实来源

`new-api` 是 token 计费和余额事实来源。

manager 不做：

- token 单价计算
- 实际扣费
- 模型计费规则维护
- token 明细落库

### 10.2 组织充值

平台管理员给组织充值 Token Credit。

流程：

1. 平台管理员选择组织。
2. 输入充值点数和备注。
3. manager 调用 `new-api` 给组织对应账号增加额度。
4. `new-api` 成功后，manager 保存充值记录。
5. 如果 `new-api` 失败，manager 不写成功记录。

充值记录包含：

- 组织 ID
- 充值点数
- 操作人
- 操作时间
- 备注
- `new-api` 引用 ID
- 成功或失败状态

充值单位只使用 Token Credit/点数。

### 10.3 成员预算

成员预算由 manager 本地维护，用于组织内部成本控制。

组织管理员可以设置：

- 成员预算额度。
- 预警阈值，例如 80%、100%。
- 超额处理策略。

超额处理策略：

- 仅预警。
- 超额自动禁用成员名下应用 api_key。

### 10.4 预算超额处理

当策略为自动禁用时：

1. manager 查询成员名下应用 api_key 的 `new-api` 用量。
2. 聚合得到成员用量。
3. 如果超过预算，禁用该成员名下应用 api_key。
4. 容器默认继续运行。
5. 应用展示 `budget_limited` 状态。
6. 写入审计日志。

组织管理员增加预算或调整策略后，可以恢复 api_key。

### 10.5 Token 统计维度

报表支持：

- 应用维度：按应用 api_key 查询。
- 成员维度：聚合成员名下应用 api_key。
- 组织维度：查询组织对应 `new-api` 账号或聚合组织下 api_key。
- 平台总量：平台管理员视角聚合所有组织。

展示字段根据 `new-api` 能力决定：

- 请求次数
- prompt tokens
- completion tokens
- total tokens
- 消耗点数
- 时间范围趋势
- 模型维度分布

统计按需查询 `new-api`。manager 不维护完整 token 明细表。

## 11. 容器运行与运维

### 11.1 容器生命周期

每个应用对应一个独立 Docker 容器。

manager 支持：

- 创建容器
- 启动容器
- 停止容器
- 重启容器
- 删除容器
- 查看容器状态
- 查看最近日志
- 查询资源使用情况

### 11.2 容器配置

容器启动时注入：

- 应用 ID
- 组织 ID
- `new-api` base URL
- 应用专用 api_key
- OpenClaw 配置目录
- AI 人设配置
- 微信插件配置
- 知识库目录
- 健康检查端口或命令

api_key 不在页面明文展示。manager 应加密保存敏感信息，或仅保存可用于查询、禁用、恢复的 key 标识。

### 11.3 运行状态判断

应用运行状态由三类信息综合得到：

- manager 业务状态。
- Docker 容器状态。
- OpenClaw 健康检查结果。

运行状态包括：

- 运行中
- 启动中
- 已停止
- 重启中
- 错误
- 未知
- 预算受限

### 11.4 增强运维能力

后台应提供：

- 启动、停止、重启应用容器。
- 查看最近日志，支持手动刷新。
- 查看 CPU、内存、网络、磁盘使用。
- 设置异常退出自动重启策略。
- 健康检查失败状态和最近失败时间。
- 高风险操作确认弹窗。
- 运行操作审计日志。

不提供：

- OpenClaw 统一代理。
- 应用外部域名分配。
- 复杂日志全文搜索。
- 告警平台。
- 多节点调度。

## 13. 外部系统接口

### 13.1 new-api

manager 依赖 `new-api` 能力：

- 创建或绑定组织对应账号。
- 查询组织余额和配额。
- 给组织充值 Token Credit。
- 创建应用专用 api_key。
- 启用、禁用、恢复 api_key。
- 查询 api_key 用量。
- 查询组织账号用量。
- 查询日志或统计报表。

如果 `new-api` 某些管理 API 不支持，需要在实施前确认替代方案。

### 13.2 Docker / Runtime

第一版依赖本机 Docker：

- 容器创建
- 容器启动
- 容器停止
- 容器重启
- 容器删除
- 容器状态
- 容器日志
- 容器资源使用
- 容器内命令执行

### 13.3 OpenClaw 与 openclaw-weixin

manager 通过容器执行 OpenClaw 操作：

- 写入应用配置。
- 写入 AI 人设。
- 配置 `new-api` api_key。
- 安装或启用微信插件。
- 执行 `openclaw channels login --channel openclaw-weixin`。
- 获取二维码。
- 查询绑定状态。
- 上传知识库文件。
- 删除知识库文件或引用。
- 查询 OpenClaw 健康状态。

## 14. 异常处理

需要显式展示并记录的异常：

- `new-api` 不可用。
- 组织对应 `new-api` 账号不存在或失效。
- api_key 创建失败。
- api_key 禁用或恢复失败。
- Docker 不可用。
- 容器创建失败。
- 容器启动失败。
- 容器异常退出。
- 健康检查失败。
- 微信插件缺失。
- 微信二维码过期。
- 微信扫码失败。
- 微信凭证失效。
- 知识库上传失败。
- 知识库删除失败。

异常展示原则：

- 用户能看到当前状态、失败原因和可执行动作。
- 管理员能看到更完整的错误详情。
- 敏感信息不展示在前端。
- 所有敏感失败写入审计日志或操作记录。

## 15. 审计日志

审计范围：

- 登录成功和失败。
- 组织创建、启用、禁用。
- 组织充值。
- 成员创建、禁用、重置密码。
- 成员预算调整。
- 预算策略变更。
- api_key 自动禁用和恢复。
- 应用创建、发布、删除。
- 容器启动、停止、重启、删除。
- 微信绑定、解绑、重试。
- 知识库文件上传、删除。
- 组织 AI 人设变更。

审计日志不可由普通业务 API 修改或删除。

## 16. 非功能需求

### 16.1 安全

- 密码必须哈希存储。
- 后端必须校验角色和组织数据范围。
- api_key 等敏感配置不能明文展示。
- 删除应用、停止容器、重置密码、充值等操作需要确认和审计。
- 禁用组织后，该组织管理员和成员不能登录。

### 16.2 可恢复性

- 创建流程支持中途退出后继续。
- 应用状态可通过后端和外部系统状态重新恢复。
- 跨系统创建失败必须有补偿策略。

### 16.3 可观测性

- 应用详情展示容器状态、健康状态、微信绑定状态、api_key 状态和最近日志。
- 平台和组织首页展示异常应用和预算风险。

### 16.4 扩展性

- 预留 Runtime Node。
- 预留多节点运行能力。
- 预留组织知识库复用能力，但当前不实现。
- 预留邀请注册能力，但当前不实现。

## 18. 已确认产品决策

- 文档目标是完整产品蓝图。
- 部署形态是先单机，预留多节点扩展。
- 每个 OpenClaw 应用最多绑定一个微信账号。
- 组织对应 `new-api` 一个账号。
- OpenClaw 应用对应 `new-api` 一个 api_key。
- 组织统一充值，成员预算由 manager 本地控制。
- 预算超额策略由组织配置：仅预警或自动禁用 api_key。
- 充值单位为 Token Credit/点数，不处理真实支付。
- 应用创建采用向导式流程。
- 微信绑定前必须创建并启动容器。
- 绑定失败后容器保持运行。
- 应用删除采用软删除，删除容器并禁用 api_key。
- 知识库为应用私有，不做组织级知识库。
- 账号体系采用账号密码登录。
- 权限采用固定三角色，不做完整 RBAC。
- Token 统计按需查询 `new-api`，manager 不保存 token 明细。
- 运行运维包含资源监控、健康检查和自动重启策略。
- AI 人设为组织级默认，组织可决定成员是否允许覆盖。
- manager 不管理 Ollama 或大模型配置。
- manager 不提供 OpenClaw 反向代理或统一流量入口。
- 知识库上传为可跳过步骤，创建后也可继续上传。
