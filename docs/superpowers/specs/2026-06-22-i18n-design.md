# 项目国际化（i18n）— 设计文档

- 日期：2026-06-22
- 范围：前端 `web/`、后端 `internal/`、hermes 运行时补丁 `runtime/hermes/`
- 触发：项目需要支持多语言。默认英文，用户可自行选择语言；登录页可选语言，登录后语言跟随用户并持久化，切换语言后相关文案实时变动。

## 目标

为整个平台引入国际化能力：

1. **管理后台前端**：全部 UI 文案可多语言，默认英文。
2. **后端返回文案**：错误信息、审计标签/详情可多语言（不把已渲染文案冻结进数据库）。
3. **hermes 运行时系统文案**：机器人对终端用户说话的语言可控（默认英文）。

支持语言：**English（默认）+ 简体中文 `zh`**。目录结构按可扩展设计，新增语言只需新增一个 locale 文件。

> 已明确排除：「可配置展示哪些渠道」这一新功能不在本期范围（用户确认暂不做）。渠道展示名如出现在错误/审计中，按下文「后端返回文案」统一处理。

## 关键决策（已与用户确认）

| 决策点 | 结论 |
|---|---|
| 翻译位置 | **混合**：所有 **UI 文案统一在前端**用一套 vue-i18n 目录翻译（含错误、审计标签/详情）；后端 i18n 目录留给「直出文案」（邮件/导出）。**当前无直出文案场景，本期不建后端 i18n 目录。** |
| 平台默认语言 | 后端配置文件新增 `i18n.default_locale`，经公开端点 `GET /api/v1/config` 暴露给前端。 |
| 登录页语言来源 | `localStorage.locale` → 平台默认（来自公开配置）→ `en` 兜底。 |
| 登录后语言来源 | `user.locale`（DB，经 `/me` 返回）为准；同时回写 localStorage。 |
| 切换语言 | 立即应用到 vue-i18n + naive-ui；写 localStorage；已登录则调后端持久化到 DB。 |
| 审计文案 | **不把已渲染中文写入数据库**；存结构化 `metadata`，前端按 `action + metadata + locale` 实时渲染，历史行回退展示存量 `detail_message`。 |
| hermes 语言来源 | 创建 app 时**快照 owner 当时的语言**；之后在应用设置里可手动改，改后触发容器重启生效。 |

## 现状（已核实）

- **前端**：Vue 3 + naive-ui + pinia + vue-router，**当前无任何 i18n 库**，文案为硬编码中文。入口 `web/src/main.ts` 装配 Pinia / Router / Vue Query。
- **后端错误**：`internal/api/apierror/response.go` 的 `ErrorResponse` 已有机器可读 `Code`（SCREAMING_SNAKE_CASE，注释明确「前端用来做 i18n」）+ 中文 `Message`。
- **审计**：`internal/service/audit_service.go` 的 `action`/`target_type`/`result`/`actor_role` **已是 code**，读取时经 `internal/service/audit_label.go` 的映射表翻译成中文。**但 `audit_logs.detail_message` 是「调用方拼好的中文短句，写入即冻结」**（例：`渠道 微信`），正是需要去冻结化的点。
- **直出文案**：后端**无**邮件 / 短信 / 导出 / 通知等直出用户文案场景。
- **登录页公开配置**：当前无通用公开配置端点；验证码靠探测 `GET /api/v1/auth/altcha-challenge` 是否 204 判断启用。
- **hermes**：自带 i18n（`agent/i18n.py` + `locales/zh.yaml`，由 `display.language` 驱动）。构建期补丁 `runtime/hermes/.../patches/patch_i18n_literals.py` 把漏翻的英文裸串**硬替换成中文**——与「默认英文」冲突，需重做。

## 架构

### 语言解析优先级

```
登录页（未登录）：localStorage.locale → 平台默认(GET /api/v1/config) → "en"
登录后：          user.locale(DB, 经 /me) → localStorage → 平台默认 → "en"
切换语言：        立即应用 vue-i18n + naive-ui → 写 localStorage → 已登录则 PATCH 持久化到 DB
```

### 核心新增件

- **前端 `useLocaleStore`（Pinia）**：持有当前 locale；负责初始化（/me 或 localStorage）与 `setLocale()`，后者联动 vue-i18n、naive-ui `NConfigProvider`、localStorage、后端持久化。
- **`users.locale` 列**：可空（null = 用平台默认）；`/me` 返回；新增持久化接口。
- **公开配置端点 `GET /api/v1/config`**：返回 `default_locale`（及 `supported_locales`）。

## Surface 1 — 前端管理后台 i18n

- **引入 `vue-i18n`**（Vue 3 生态事实标准），在 `main.ts` 装配。
- **locale 文件**：`web/src/i18n/locales/{en,zh}.ts`，key 按页面/领域分组（如 `apps.list.title`）。`en` 为默认与兜底。
- **naive-ui 语言联动**：`NConfigProvider` 的 `locale` / `dateLocale` 跟随当前 locale 切 `enUS/zhCN` 与 `dateEnUS/dateZhCN`，使组件内置文案与日期选择器一起变。
- **存量中文迁移**：硬编码中文逐页换成 `t()`。按模块分批：login / dashboard / apps / org / audit / knowledge / usage / platform / skills 等。
- **语言选择器**：
  - 登录页一个（写 localStorage，未登录）。
  - 登录后放顶栏/用户菜单（写 DB + localStorage + 实时切换）。

## Surface 2 — 后端返回文案

- **`users` 加 `locale` 列**（migration + sqlc 查询）。
- **接口**：
  - `GET /api/v1/config`（公开）返回 `default_locale`（+ `supported_locales`）。
  - `/me` 返回 `locale`。
  - 持久化接口（`PATCH /me/locale` 或并入 profile 更新），写入 `users.locale`。
- **错误**：保留 `Code`，前端按 Code 翻译；后端 `Message` 保留作兜底，不阻塞。
- **审计去冻结化**：
  - 停止往 `detail_message` 写拼好的中文；改为把详情所需字段写入结构化 `metadata`。
  - 前端按 `action + metadata + locale` 实时拼详情。
  - **历史行兼容**：前端「有结构化 metadata 就本地化渲染，否则回退展示存量 `detail_message`」。
  - `actionLabels` / `targetTypeLabels` / `resultLabels` / `actorRoleLabels` 等映射从后端移到前端目录；后端继续只返回 raw code（本就如此），冗余的 `*_label` 字段逐步下线。

> 注：`audit_label.go` 的标签映射移到前端后，需评估是否仍需保留后端 `*_label` 字段以兼容；倾向「先保留、新前端不依赖、后续清理」，避免破坏接口契约。

## Surface 3 — hermes 运行时文案

- **app 新增 `locale` 字段**：创建时快照 owner 当时语言；应用设置里可手动改，改后触发**容器重启**生效（hermes 重新渲染语言需重启）。
- 该值写入生成配置的 hermes `display.language`（经 `internal/integrations/hermes/app_input.go` / manifest）。
- **重做强制中文补丁**：`patch_i18n_literals.py` 现把漏翻裸串硬替换成中文，与默认英文冲突。改为「把漏翻裸串包进 hermes 原生 `t()` 并在 locale yaml 补 key（en 保留原文 + zh.yaml 补译文）」，让 `display.language` 正确驱动两种语言。**这是最贴近上游、风险最高的一块，排到最后阶段。**

## 分阶段落地

每阶段后续各自走 plan → 实现 → 验证。

| 阶段 | 内容 | 交付物 |
|---|---|---|
| **P1 地基** | vue-i18n 装配、`useLocaleStore`、naive-ui 语言联动、`users.locale` 列 + `/me` + 持久化接口 + 公开配置端点、登录页/顶栏两个语言选择器 | 切换链路打通：能切语言、跟随用户、刷新/换设备保持 |
| **P2 前端文案迁移** | 存量中文 → `t()`，按模块分批；en/zh 两套目录补齐 | 后台界面整体可多语言 |
| **P3 审计去冻结 + 标签前移** | `detail_message` 结构化、label 映射移前端、老行兼容回退 | 切语言后历史审计也跟着变 |
| **P4 hermes 运行时** | app `locale` 字段（创建快照 + 手动改 + 重启）、写入 `display.language`、重做 i18n 补丁 | 机器人按 owner 语言说话 |

**依赖**：P1 是地基，必须最先；P2 / P3 可并行；P4 独立、风险最高，放最后。

## 测试策略

- **前端**：vitest 覆盖 `useLocaleStore`（优先级解析、切换联动、持久化调用）与选择器组件；E2E（playwright）覆盖「登录页切语言 → 登录 → 跟随 DB → 切换实时变」主链路。
- **后端**：Go 测试（testify，遵循 AGENTS.md）覆盖 `users.locale` 持久化、`/me` 返回、审计写入改为结构化 metadata。
- **防回潮**：加检查项识别新增硬编码中文裸串，避免迁移完又混入。
- **交付前**：按 AGENTS.md 用**真实浏览器**全面验证（curl 不能替代前端验证）。

## OpenAPI 同步

新增 / 修改 handler（`/api/v1/config`、`/me` locale、持久化接口）后，按 AGENTS.md 跑 `make openapi-gen` + `make web-types-gen`，保持 `openapi/openapi.yaml` 与 `web/src/api/generated.ts` 与代码同步。
