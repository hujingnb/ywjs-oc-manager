# AICC 工作台导航重构设计

- 日期：2026-07-11
- 状态：设计确认稿
- 关联设计：
  - `docs/superpowers/specs/2026-07-08-online-customer-service-design.md`
  - `docs/superpowers/specs/2026-07-09-aicc-implementation-design.md`

## 1. 背景

当前 `/aicc-console` 已作为独立客服子系统，不再嵌入主后台实例页面。但现有工作台左侧菜单把「会话」「知识库」「线索」「分析」「设置」作为外壳级导航展示，而右侧页面内部又围绕「智能体」和配置分区组织内容。

这造成两个问题：

- 「会话」「线索」等更像当前智能体下的内容模块，不像 AICC 子系统的左侧主菜单。
- 外层左侧导航和右侧智能体列表、内部 tab 形成多层导航，用户容易不清楚当前是在切模块、切智能体，还是切配置分区。

用户已确认采用 **B/F 合并方案**：顶部模块导航 + 紧凑当前智能体上下文条。

## 2. 设计目标

本次重构只调整 AICC 工作台管理端的信息架构和页面外壳，不改变 AICC 后端数据模型、API 契约和公开访客聊天页。

目标：

1. 左侧不再展示「会话/线索/知识库/分析/设置」这类右侧内容模块。
2. AICC 子系统的主功能区使用顶部模块导航承载。
3. 当前智能体上下文始终可见，避免用户切换模块后不知道正在查看哪个智能体。
4. 智能体列表保留在内容区，作为当前模块的业务对象选择，不再承担外壳级主菜单职责。
5. 现有深链 `/aicc-console/sessions`、`/aicc-console/knowledge`、`/aicc-console/leads`、`/aicc-console/analytics`、`/aicc-console/settings` 继续可用。

非目标：

- 不新增语音客服页面。
- 不重做访客端公开聊天页。
- 不改变组织 AICC 开通校验、登录校验和角色权限。
- 不重写 AICC 管理端 API。

## 3. 最终方案

采用三段式顶部结构 + 内容区左右分栏。

### 3.1 顶部品牌栏

品牌栏位于页面最上方，包含：

- AICC 标识和标题。
- 语言切换。
- 返回概览按钮。

品牌栏只表达「当前在 AICC 子系统内」，不放业务模块列表。

### 3.2 顶部模块导航

品牌栏下方展示 AICC 模块导航：

- 接待台
- 会话
- 线索
- 知识库
- 分析
- 设置

模块导航是 AICC 子系统主导航，替代当前左侧外壳菜单。点击后仍走现有路由：

- 接待台：`/aicc-console`
- 会话：`/aicc-console/sessions`
- 线索：`/aicc-console/leads`
- 知识库：`/aicc-console/knowledge`
- 分析：`/aicc-console/analytics`
- 设置：`/aicc-console/settings`

路由语义保持稳定，便于刷新、收藏和后续权限/埋点扩展。

### 3.3 当前智能体上下文条

模块导航下方展示紧凑的当前智能体上下文条，包含：

- 当前智能体名称。
- 运行状态。
- 公开入口状态。
- 关键配置摘要，例如数据保留天数。
- 切换智能体入口。
- 新建智能体入口。

上下文条只展示摘要，不把多个智能体按钮全部铺在顶部，避免智能体数量增加后顶栏拥挤。

当尚未选择智能体时，上下文条展示「未选择智能体」和「新建智能体」入口。

### 3.4 内容区

内容区保持左右分栏：

- 左侧为智能体列表和当前智能体概况。
- 右侧为当前模块内容。

智能体列表属于内容区，不属于外壳级主菜单。选择不同智能体后，右侧当前模块内容按新智能体刷新。

各模块内容职责：

- 接待台：智能体基础配置、投放入口、运营配置和关键状态。
- 会话：当前智能体会话列表、筛选和会话详情。
- 线索：当前企业或当前智能体线索列表、未读状态和导出入口。
- 知识库：当前智能体检索来源配置。
- 分析：当前智能体或企业维度运营统计。
- 设置：偏系统级和低频配置；若与接待台配置重复，实施时应收敛，不重复展示同一组 tab。

## 4. 与 B / F 的取舍

### 4.1 相比 B

B 是单层顶部模块导航，页面最简洁，内容区空间最大。但 B 没有稳定展示当前智能体上下文，用户切换到会话、线索或分析时，需要在内容区重新确认自己正在看哪个智能体。

合并方案保留 B 的顶部模块导航，同时增加一条轻量智能体上下文条，降低迷路概率。

### 4.2 相比 F

F 把模块和智能体都放在顶部，切换很直接。但智能体数量增加后，顶部智能体按钮会拥挤，并且占用较多垂直空间。

合并方案不在顶部展开全部智能体，只展示当前智能体摘要和切换入口。完整智能体列表仍放在内容区左侧。

## 5. 移动端与响应式

桌面端优先保证信息密度和操作效率。

移动端规则：

- 顶部模块导航横向滚动。
- 当前智能体上下文条压缩为名称 + 状态 + 切换按钮。
- 内容区左右分栏改为上下结构，智能体列表可折叠。

本期只需保证现有桌面管理端可用；移动端不作为主要验收场景，但样式不能出现明显文本重叠或不可访问按钮。

## 6. 国际化

所有用户可见文案继续走现有 i18n 体系，不在 Vue 模板中新增硬编码中文或英文。

需要覆盖的文案包括：

- 模块导航标签。
- 当前智能体上下文条标签和空状态。
- 切换智能体、新建智能体等按钮。
- 响应式折叠或下拉入口文案。

## 7. 测试与验证

自动化测试需要覆盖：

- `/aicc-console` 外壳不再渲染左侧 `n-layout-sider` 模块菜单。
- 顶部模块导航包含接待台、会话、线索、知识库、分析、设置。
- 点击模块导航进入对应 `/aicc-console/*` 路由。
- 当前智能体上下文条在有智能体、无智能体、加载中三种状态下展示合理。
- 企业未开通 AICC 时，仍不会挂载子页面并会回到概览。
- i18n 中文、英文 key 完整。

真实浏览器验证需要覆盖：

- 企业管理员从概览页进入 `/aicc-console`。
- 顶部模块导航切换全部模块。
- 切换智能体后，当前上下文条和右侧内容同步更新。
- 返回概览正常。
- 浏览器 console 无 error/warn，AICC 相关 XHR/fetch 状态正常。

## 8. 实施边界

建议实施时优先改前端结构：

- `AICCConsoleLayout.vue`：移除左侧外壳菜单，改为顶部品牌栏、模块导航和智能体上下文条。
- `AICCManagerPage.vue` 及相关子页面：避免顶部模块和内部 tab 重复表达同一层级。
- `AICCConsoleLayout.spec.ts`：更新外壳导航测试。
- `aicc` i18n 语言包：补齐新增文案。

除非实现中发现确需调整，否则不修改后端、数据库迁移、OpenAPI 和生成类型。

## 9. 验收标准

完成后应满足：

- AICC 工作台不再出现左侧「会话/线索/知识库/分析/设置」外壳菜单。
- 用户能在顶部清晰切换 AICC 模块。
- 用户始终能看到当前智能体上下文。
- 智能体列表留在内容区，不再和主导航混淆。
- 原有深链和企业开通保护保持正常。
- 自动化测试、类型检查、构建和真实浏览器验证通过。
