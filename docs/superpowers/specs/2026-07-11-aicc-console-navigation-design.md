# AICC 工作台导航重构设计

- 日期：2026-07-11
- 状态：设计确认稿
- 关联设计：
  - `docs/superpowers/specs/2026-07-08-online-customer-service-design.md`
  - `docs/superpowers/specs/2026-07-09-aicc-implementation-design.md`

## 1. 背景

当前 `/aicc-console` 已作为独立客服子系统，不再嵌入主后台实例页面。但现有工作台左侧菜单把「会话」「知识库」「线索」「分析」「设置」作为外壳级导航展示，而右侧页面内部又围绕「智能体」和配置分区组织内容。

这造成两个问题：

- 「会话」「线索」等更像当前智能体下的内容模块，不像 AICC 子系统的左侧主菜单。
- 外层左侧导航和右侧智能体列表、内部 tab 形成多层导航，用户容易不清楚当前是在切模块、切智能体，还是切配置分区。

用户已确认采用 **顶部智能体选择 + 左侧模块菜单** 的最终方案。此前 B/F 合并方案作为中间设计已被替换：智能体是工作对象，应固定在顶部；会话、线索、知识库、分析、设置是当前智能体下的功能模块，应放在左侧菜单。

## 2. 设计目标

本次重构只调整 AICC 工作台管理端的信息架构和页面外壳，不改变 AICC 后端数据模型、API 契约和公开访客聊天页。

目标：

1. 顶部作为全局工作对象区，只保留当前智能体选择、新建智能体和摘要状态。
2. 左侧展示「接待台/会话/线索/知识库/分析/设置」功能菜单，表达当前智能体下的功能模块。
3. 当前智能体上下文始终可见，避免用户切换模块后不知道正在查看哪个智能体。
4. 内容区不再重复展示智能体列表；顶部选择智能体后，内容区直接展示当前模块内容。
5. 现有深链 `/aicc-console/sessions`、`/aicc-console/knowledge`、`/aicc-console/leads`、`/aicc-console/analytics`、`/aicc-console/settings` 继续可用。

非目标：

- 不新增语音客服页面。
- 不重做访客端公开聊天页。
- 不改变组织 AICC 开通校验、登录校验和角色权限。
- 不重写 AICC 管理端 API。

## 3. 最终方案

采用顶部工作对象区 + 左侧功能菜单 + 右侧内容区。

### 3.1 顶部品牌与智能体栏

顶部栏位于页面最上方，包含：

- AICC 标识和标题。
- 当前智能体选择器。
- 当前智能体摘要：运行状态、公开入口状态、数据保留天数。
- 新建智能体入口。
- 语言切换。
- 返回概览按钮。

顶部栏只表达「当前在 AICC 子系统内」和「当前操作哪个智能体」，不放功能模块菜单。

智能体选择是全局唯一入口。选择不同智能体后，右侧当前模块内容按新智能体刷新。

### 3.2 左侧功能菜单

左侧展示 AICC 功能菜单：

- 接待台
- 会话
- 线索
- 知识库
- 分析
- 设置

这些菜单是当前智能体下的功能模块，不再和智能体列表混在内容区。点击后仍走现有路由：

- 接待台：`/aicc-console`
- 会话：`/aicc-console/sessions`
- 线索：`/aicc-console/leads`
- 知识库：`/aicc-console/knowledge`
- 分析：`/aicc-console/analytics`
- 设置：`/aicc-console/settings`

路由语义保持稳定，便于刷新、收藏和后续权限/埋点扩展。

### 3.3 当前智能体空态

当尚未选择智能体时，顶部智能体选择区展示「未选择智能体」，状态摘要显示保存后生成等空态文案。内容区进入新建智能体表单，不展示旧智能体残留信息。

### 3.4 内容区

内容区只展示当前菜单对应的业务内容，不再展示智能体列表。

各模块内容职责：

- 接待台：智能体基础配置、投放入口、运营配置和关键状态。
- 会话：当前智能体会话列表、筛选和会话详情。
- 线索：当前企业或当前智能体线索列表、未读状态和导出入口。
- 知识库：当前智能体检索来源配置。
- 分析：当前智能体或企业维度运营统计。
- 设置：偏系统级和低频配置；若与接待台配置重复，实施时应收敛，不重复展示同一组 tab。

## 4. 与 B / F 的取舍

### 4.1 相比 B/F 合并方案

B/F 合并方案把功能模块放在顶部，并把当前智能体摘要也放在顶部。实际浏览器视觉稿验证后，用户反馈“顶部选择智能体了，内容区就不需要了”，并确认功能菜单更适合放回左侧。

最终方案的判断是：顶部应只负责全局工作对象；左侧应负责当前对象下的功能模块。这样层级更清楚，内容区也不会同时出现顶部智能体选择和内容区智能体列表。

### 4.2 色系

最终视觉稿需要与主站色系保持一致，使用现有全局 CSS 变量：

- 主色：`--color-brand`（当前为 `#ff6a00`）。
- 主色浅底：`--color-brand-soft`。
- 页面背景：`--color-page`。
- 卡片背景：`--color-surface`。
- 边框/分割线：`--color-border`、`--color-divider`。

不再使用独立青绿色作为 AICC 工作台主色。

## 5. 移动端与响应式

桌面端优先保证信息密度和操作效率。

移动端规则：

- 顶部智能体选择区压缩为上下结构。
- 左侧功能菜单在窄屏下改为横向滚动菜单。
- 内容区保持单列展示。

本期只需保证现有桌面管理端可用；移动端不作为主要验收场景，但样式不能出现明显文本重叠或不可访问按钮。

## 6. 国际化

所有用户可见文案继续走现有 i18n 体系，不在 Vue 模板中新增硬编码中文或英文。

需要覆盖的文案包括：

- 左侧功能菜单标签。
- 当前智能体上下文条标签和空状态。
- 切换智能体、新建智能体等按钮。
- 响应式折叠或下拉入口文案。

## 7. 测试与验证

自动化测试需要覆盖：

- `/aicc-console` 外壳渲染左侧工作区功能菜单，但不使用旧 `n-layout-sider` 嵌套外壳。
- 左侧功能菜单包含接待台、会话、线索、知识库、分析、设置。
- 点击左侧功能菜单进入对应 `/aicc-console/*` 路由。
- 顶部智能体选择区在有智能体、无智能体、加载中三种状态下展示合理。
- 内容区不再重复渲染智能体列表。
- 企业未开通 AICC 时，仍不会挂载子页面并会回到概览。
- i18n 中文、英文 key 完整。

真实浏览器验证需要覆盖：

- 企业管理员从概览页进入 `/aicc-console`。
- 顶部选择不同智能体后，右侧内容同步更新。
- 左侧功能菜单切换全部模块。
- 内容区不出现智能体列表。
- 返回概览正常。
- 浏览器 console 无 error/warn，AICC 相关 XHR/fetch 状态正常。

## 8. 实施边界

建议实施时优先改前端结构：

- `AICCConsoleWorkspace.vue`：把功能模块导航从顶部移动到左侧，把智能体选择固定在顶部。
- `AICCManagerPage.vue` 及相关子页面：移除内容区智能体列表，消费顶部选中的智能体。
- `AICCConsoleLayout.spec.ts`：更新外壳导航测试。
- `aicc` i18n 语言包：补齐新增文案。

除非实现中发现确需调整，否则不修改后端、数据库迁移、OpenAPI 和生成类型。

## 9. 验收标准

完成后应满足：

- AICC 工作台左侧只作为功能菜单，不再与内容区智能体列表形成双重选择。
- 用户在顶部清晰切换当前智能体。
- 用户在左侧清晰切换 AICC 功能模块。
- 内容区不再出现智能体列表。
- 原有深链和企业开通保护保持正常。
- 自动化测试、类型检查、构建和真实浏览器验证通过。
