# 公开客服聊天消息发送时间设计

## 目标

在公开客服聊天页的每条消息气泡下方展示访客浏览器本地时区的发送时间，格式固定为 `HH:mm`。

## 范围

- 修改 `web/src/pages/aicc/PublicAICCChatPage.vue` 的消息内存模型、时间转换和气泡渲染。
- 修改 `web/src/pages/aicc/PublicAICCChatPage.spec.ts`，覆盖恢复历史消息及新发送消息的时间显示。
- 不修改后端接口、OpenAPI 契约或生成类型。

## 设计

页面内的 `ChatMessage` 增加可选发送时间字段。会话恢复时，直接保留 API 返回的 `AICCMessage.created_at`；访客提交消息及收到助手回复时，记录当前浏览器时间。这样消息无需等待页面刷新即可显示时间，而刷新恢复后会使用服务端保存的创建时间。

新增一个仅用于展示的格式化函数：有效时间通过浏览器本地时区格式化为两位小时和分钟；缺失或无效时间返回空字符串，且不渲染时间元素，避免显示 `Invalid Date`。时间元素放在消息气泡之后，沿气泡所属一侧对齐，使用弱化文字样式，不影响现有文字和图片消息布局。

## 验证

先添加前端单元测试并确认其在未实现前失败；实现后运行该组件测试、前端类型检查和生产构建。最后在本地真实浏览器中发送消息，确认访客消息与客服回复均在气泡下方显示 `HH:mm`，并刷新页面确认历史消息仍展示时间。

## 非目标

- 不显示日期、秒数或时区名称。
- 不调整后台会话记录页的既有时间展示。
- 不为即时消息增加后端返回的时间字段。
