# 用户手册全量重核对同步 · 设计

## 背景与目标

`docs/user-manual.md` 最近一次更新于 2026-06-05（commit `e9bcb224`），此后约一个月内落地 769 个提交，产品发生结构性变化，手册已严重滞后。本设计定义一次**全量重核对重写**：逐页面 / 逐路由把手册与当前产品核对一遍，删除已下线内容、补齐所有新页面与 tab、修正渠道与登录字段等描述，使交付后手册与前端完全一致。

手册定位不变：**面向平台管理员 / 企业管理员 / 企业成员三类角色，说明各自在 manager 前端能做什么**（页面入口、典型操作步骤、权限边界）。

## 范围

- **范围内**：`docs/user-manual.md` 的整篇重写。
- **范围外**：其他文档（architecture / product-design 等）不改；不改任何代码；不新增功能。

## 已确认的关键决策

1. **档位**：全量重核对重写（非增量补丁）。
2. **真相来源**：以阅读前端代码为主（`web/src` 下页面 / tab 组件与 `web/src/app/router.ts` 路由及角色边界）；不依赖本地环境浏览器走查。个别交互细节存疑处在代码中确认。
3. **内容边界原则**：**只写当前前端可见且可用的功能**。
   - 4 个渠道（微信 / 企业微信 / 飞书 / 钉钉）全部写入——均已接入渠道 tab。
   - 对话 tab 写入。
   - 对话页语音输入**不写**——当前被注释禁用（`AppConversationsTab.vue` 中 `VoiceInputButton` 已注释，保留实现待后续开启）。
4. **组织方式**：沿用「按角色分章」结构（方案 A），原地逐节重写；仅对「技能体系」这类真正跨三角色的新子系统，采用一段公共说明 + 各角色权限注记，避免三处全量复制（吸收方案 C 的一点）。

## 结构差异总览（旧 → 新）

以当前 `router.ts` 为准，与旧手册的主要差异：

**已下线但旧手册仍在描述（须删除）**
- 旧 §1.3「运行节点」`/runtime-nodes`：节点概念已随 k8s 迁移整体删除，无此路由——整章删除。
- §2.3「概览」中的 "Runtime Node" 字段引用——一并移除。
- 旧 `/org/persona` 独立「企业 AI 人设」页：路由已不存在；persona 逻辑现出现在 `AssistantVersionsPage`（助手版本）与 `OrganizationsPage`（企业管理）中——旧 §2.4 / §3.4 需改写为新落点。

**新增、旧手册完全未覆盖**
- 平台侧：`/console`（取代 `/platform/dashboard`）、`/platform/permissions`（权限管理）、`/platform/skills`（平台技能库）、`/platform/custom-skills`（定制技能工单）、`/platform/web-publish-config`（web 发布配置）、`/published-sites`（已发布站点）、独立充值页 `/platform/organizations/:orgId/recharge`。
- 企业侧：`/org-console`（企业控制台）、`/balance`（企业余额）。
- 全角色：`/skills`（成员技能页）、`/skill-tickets/:id`（技能工单详情）。
- 实例详情 tab 由 6 个扩为 10 个：新增 看板(kanban) / 定时任务(cron) / 技能(skills) / 对话(conversations)；「运行时(runtime)」tab 现**仅平台管理员**可见。
- 渠道由「仅微信」扩为 4 个渠道。

**描述需更新**
- 登录字段命名统一为「账号 / 姓名」（近期 i18n 提交）；补充界面语言切换（中 / 英）。
- 登录后跳转改为 `/console`（平台）或角色首页。
- 概览新增「平台提示词已更新需重启」横幅；实例状态新增 restarting / runtime_phase 维度。
