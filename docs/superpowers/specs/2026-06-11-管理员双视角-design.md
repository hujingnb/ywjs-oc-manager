# 企业管理员双视角(org_admin dual-perspective)设计文档

> 日期:2026-06-11
> 状态:设计已确认,待编写实现计划

## 背景与目标

「定制技能」功能上线后暴露一个缺口:**企业管理员(org_admin)没有属于自己的实例市场入口**。当平台交付一个「仅管理员可见」的定制技能时,org_admin 在界面上看得到这个技能,却**没有地方把它装进一台实例**——因为 org_admin 默认没有个人实例,而把它装进某个成员实例又违背「仅管理员可见」的语义。

本功能给 org_admin 引入**视角切换**:

- **企业管理视角**:管理整个企业(现有的管理菜单)。
- **我的实例视角**:管理 org_admin 自己那一台 Agent,菜单与企业成员一致。

org_admin 可以给自己创建实例;在「我的实例视角 › 技能 › 市场」里把「仅管理员可见 / 整企业可见」的定制技能装进自己的实例,语义自洽,不污染成员实例。

## 关键前提:后端已就绪(零改动)

设计阶段已读码确认,后端**完全支持**本功能,无需任何改动(无新端点、无迁移、无权限改动)。具体事实:

| 能力 | 现状 | 代码位 |
|---|---|---|
| 列实例返回 admin 自有实例 | `ListByOrg` 对 org_admin **不按 owner 过滤**,返回本企业全部实例(含 admin 自己拥有的) | `internal/service/app_service.go:124-154`;`internal/store/queries/apps.sql:200-207` |
| admin 给自己建实例 | `CanCreateAppForMember` 允许 org_admin 给本企业**任意成员(含自己)** 建实例;owner_user_id 取路径参数 userId,可为 admin 自己 | `internal/auth/authorizer.go:227-239`;`internal/service/onboarding_service.go:270-342` |
| admin 访问/管理自有实例 | `CanViewApp / CanManageApp / CanSwitchAppVersion` 对 org_admin 访问本企业任意实例(含自有)及其子资源(渠道/技能/知识库)全部放行 | `internal/auth/authorizer.go:89-142` |
| 一人一实例约束 | 唯一约束 `uk_apps_owner_active`(基于 `owner_active_key` 生成列)保证每个 owner 至多一个未删除实例,**不区分角色** | `internal/migrations/000001_baseline.up.sql:109-163` |

> 注:此前记忆中「`GET /organizations/:orgId/apps` 给 org_admin 只返回成员实例、不含 admin 自己的 app」是**误诊**。真实原因是当时的 admin 账号压根没建过实例,而非 API 过滤。`useOwnApp` 已能从该列表正确筛出 admin 自有实例。

因此本功能**纯前端**实现。

## 详细设计

### 1. 导航模型与视角切换

- 顶栏(`DashboardLayout` header,挨着「控制台」标题处)放一个分段切换器 `[企业管理 | 我的实例]`。
- 切换器**仅 org_admin 可见**;platform_admin 与 org_member 不渲染切换器,导航与现状完全一致,行为不变。
- **企业管理视角**菜单(即现有 org_admin 菜单去掉独立「技能」项):
  - 总览 / 成员 / 实例 / 企业知识库 / 账户余额 / 审计 / 用量
- **我的实例视角**菜单(与 org_member 一致,由 admin 自己的 appId 驱动):
  - 总览 / 渠道 / 工作目录 / 个人知识库 / 企业知识库 / 技能 / 任务 / 定时任务 / 用量
- **「技能」菜单迁移**:org_admin 现有的独立「技能」菜单项,从企业管理视角**移入「我的实例」视角**。企业管理视角不再单挂技能。
- **默认视角**:org_admin 登录后默认落「企业管理视角」。
- **视角持久化**:当前视角写入 `localStorage`,刷新页面不跳回默认;退出登录时清除(避免换账号串视角)。

### 2. 建实例与空状态

- **我的实例视角 + 无实例**:实例类页面(总览/渠道/工作目录/任务/定时任务等,这些页面本质需要一个 appId)显示**空状态** +「去成员页创建」按钮。
  - 按钮跳转到企业管理视角的「成员」页,由 admin 使用自己那行已有的「为该成员创建实例」完成创建(复用现有流程,零新建创建逻辑)。
  - 创建完成后,`useOwnApp` 会查到该实例,我的实例视角各页面正常驱动。
- **技能页例外**:即使无实例,「我的实例视角 › 技能」仍渲染定制技能工单面板(沿用现有 `OrgSkillsPage` 的 `SkillTicketPanel` + 无实例 banner 行为),org_admin 始终能提交/跟踪定制技能工单。建了自己实例后,技能页展示完整的「已安装 / 技能市场 / 定制技能」三 tab,可在市场安装「仅管理员可见 / 整企业可见」的定制技能到自己实例——**这是本功能要解决的原始痛点**。

### 3. 前端改动面

- `web/src/layouts/DashboardLayout.vue`:
  - 引入视角状态(`localStorage` 持久化,默认 `企业管理`),仅在 org_admin 时生效。
  - 顶栏渲染分段切换器(仅 org_admin)。
  - `menuOptions` 按 **角色 × 视角** 分支:org_admin 企业管理视角输出管理菜单(去技能);org_admin 我的实例视角输出成员同款菜单(由 `useOwnApp` 的 appId 驱动,复用现有 `memberAppTabKey/memberAppTabPath` 那套逻辑,使其同时服务 admin)。
  - `activeKey` 适配我的实例视角的 `/apps/:appId/:tab` 高亮(复用现有成员高亮逻辑)。
  - 退出登录时清除视角状态。
- `useOwnApp`(已就绪):驱动我的实例视角的 appId。我的实例视角**复用 org_member 现有路由/页面**(`/apps/:appId/*`、`/skills` → `OrgSkillsPage`),不新建页面。
- 新增「无实例空状态」展示 + 跳成员页的交互(可放在实例类页面的无 app 兜底处,或一个小型共享组件)。
- 路由守卫/可访问性:确认 org_admin 访问 `/apps/:ownAppId/*` 不被前端路由限制拦截(后端已放行)。

### 4. 不做的事(YAGNI)

- 不为 admin 建实例新增专用端点或表单(复用成员页创建)。
- 不改任何后端代码、权限谓词、数据库。
- 不给 platform_admin / org_member 加视角切换。
- 不做视角间的数据隔离(同一 admin 两视角看的是同一份数据的不同子集,天然由菜单与页面区分)。

## 测试与验证

### 前端单元测试(vitest)
- `menuOptions`:org_admin 在企业管理视角输出管理菜单且**不含**技能;在我的实例视角输出成员同款菜单且**含**技能;org_member 与 platform_admin 菜单不受影响。
- 切换器仅在 org_admin 渲染;org_member / platform_admin 不渲染。
- 视角状态 `localStorage` 持久化与默认值;退出登录清除。

### 真实浏览器三态验证(三角色)
- **org_admin 无实例**:登录默认企业管理视角 → 切「我的实例」→ 实例类页面显示空状态 → 点「去成员页创建」跳成员页 → 用自己行建实例 → 回我的实例视角各页面正常;无实例时技能页仍可提交工单。
- **org_admin 有实例**:我的实例视角全菜单可用(总览/渠道/.../技能);技能页三 tab 完整;在市场安装一个「仅管理员可见」定制技能到自己实例并生效;切回企业管理视角管理菜单正常。
- **回归**:org_member 与 platform_admin 登录后**无切换器**,菜单与行为与现状一致。

### 交付前检查
- 工作区无无关改动;OpenAPI/生成类型无需变更(无后端改动)。
- 浏览器验证留逐项证据(遵循项目验证标准)。

## 影响面小结

- 后端:无。
- 数据库:无。
- 前端:`DashboardLayout.vue` 为主,加视角状态/切换器/菜单分支;`useOwnApp` 复用;复用成员现有页面;一个无实例空状态交互。
- 风险:主要是导航分支逻辑的回归(勿影响 org_member/platform_admin),由单测 + 浏览器回归覆盖。
