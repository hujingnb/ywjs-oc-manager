# 移除后台环境与角色标识设计

## 背景

后台顶栏当前固定展示“本地调试环境 · {role}”。该文案不会根据部署环境变化，在线上站点同样显示“本地调试环境”，且直接暴露内部角色代码，容易误导用户。

## 目标

- 从后台顶栏彻底移除环境与角色标识。
- 保留“控制台”标题、视角切换、语言切换、API 状态、使用手册和刷新功能。
- 清理仅服务于该标识的计算属性及中英文国际化文案。

## 实现范围

在 `DashboardLayout.vue` 中删除渲染环境标识的节点和 `environmentLabel` 计算属性；在中英文布局语言包中删除 `envLabel` 与 `envLabelWithRole`。不修改登录用户结构、角色权限判断、路由或后端授权逻辑。

## 验证

- 在现有 `DashboardLayout.spec.ts` 中增加回归测试，确认顶栏不再展示环境文案和内部角色代码，同时仍展示“控制台”。
- 先运行新增测试并确认其因旧标识仍存在而失败，再执行实现并运行定向测试。
- 使用本地真实浏览器打开后台，确认顶栏布局和其余操作入口正常。
