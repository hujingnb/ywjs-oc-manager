# 首页登录页面居中适配 — 设计文档

- 日期：2026-06-03
- 状态：已确认，待实现
- 作者：hujing + Codex

## 背景与目标

当前登录页由 `web/src/layouts/AuthLayout.vue` 承载：外层 `.auth-stage` 同时负责背景装饰、两列
布局、视口内边距和响应式切换；左侧为平台介绍 hero，右侧为 `RouterView` 渲染的登录表单。
桌面大屏下两列内容依赖外层 grid 与 `justify-self` 分别贴近左右侧，整体视觉重心不够居中。

本次目标是让首页登录页面在不同窗口尺寸下动态适应，并使左侧介绍和右侧登录卡片作为一个
整体在视口中居中展示。登录表单逻辑、背景动画和文案内容保持不变。

## 已确认决策

1. **采用显式内容容器方案。** 在 `AuthLayout.vue` 中新增 `.auth-content` 包住 `.auth-hero`
   与 `.auth-login-shell`，让内容布局职责从 `.auth-stage` 中拆出来。
2. **保留现有品牌展示。** 不移除左侧 hero，不弱化背景动画，也不改变登录卡片内部字段。
3. **整体居中而非单卡片居中。** 桌面端左右两块作为一组居中，避免大屏下内容发散到两侧。
4. **高度不足时允许滚动。** 登录表单不能被裁切；小高度窗口优先保证可访问性和可操作性。
5. **范围限定前端布局。** 只修改 `AuthLayout.vue` 模板结构和 scoped CSS，不改路由、认证 store、
   登录请求或 OpenAPI。

## 方案比较

### 选定方案：新增 `.auth-content` 显式布局容器

`.auth-stage` 继续作为全屏舞台，负责背景层、颜色变量、滚动和基础视口约束；新增
`.auth-content` 负责 hero 与登录卡片的 grid 布局、整体最大宽度、水平居中和垂直居中。

优点：

- 背景职责和内容布局职责分离，后续调整响应式时边界更清楚。
- 能精确控制左右内容作为一个整体居中，而不是分别通过 `justify-self` 修补位置。
- 保留现有视觉资产和登录逻辑，改动集中。

代价：

- 需要调整模板层级，视觉回归范围略大于只改现有 CSS。
- 现有 `.auth-stage`、`.auth-hero`、`.auth-login-shell` 的部分 grid 样式需要迁移到
  `.auth-content`，实现时要避免遗留冲突。

### 备选方案：只改 `.auth-stage` grid

保留当前模板结构，通过 `.auth-stage` 设置整体最大宽度、居中和 padding。改动更少，但
`.auth-stage` 同时承担背景和内容排版，后续窗口适配规则会继续耦合在一个选择器中。

### 不采用方案：绝对定位或 transform 居中

可以快速居中视觉区域，但在窗口高度不足、移动端滚动、浏览器地址栏动态高度变化时更容易
裁切表单或产生滚动异常，因此不采用。

## 布局设计

### 桌面端

- `.auth-stage`：
  - 保持 `position: relative`、背景层和 `isolation` 等舞台能力。
  - 使用 `min-height: 100dvh` 适配现代移动和桌面浏览器动态视口；必要时保留 `100vh`
    兜底。
  - 使用 grid 或 flex 将 `.auth-content` 放到页面中心。
  - 内边距使用 `clamp()`，在大屏不过度贴边，在窄屏不挤压表单。
  - 横向保持隐藏溢出，纵向允许滚动，避免小高度窗口裁切登录表单。
- `.auth-content`：
  - 使用两列 grid：左侧 `minmax(0, 1.1fr)`，右侧保持登录区域固定上限。
  - 设置整体 `width: min(100%, 1280px)`，通过最大宽度限制大屏横向扩散。
  - 设置 `align-items: center`，使 hero 和登录卡片围绕同一垂直中心排列。
  - 设置 `gap: clamp(28px, 4vw, 56px)`，随窗口宽度平滑变化。
- `.auth-hero`：
  - 继续承载介绍文案和能力卡片。
  - 去除依赖外层 grid 的 `justify-self: end`，改由 `.auth-content` 管理整体位置。
- `.auth-login-shell`：
  - 保持登录卡片宽度上限和装饰边框。
  - 在右列内对齐到合适位置，不再把整块推到页面右侧。

### 中等和窄屏

在现有 `980px` 断点附近切换为单列：

- `.auth-content` 改为一列 grid，宽度撑满可用空间并居中。
- `.auth-hero`、`.auth-login-shell` 均水平居中。
- `.auth-hero` 的 `min-height` 继续归零，避免移动端首屏被介绍内容撑得过高。
- `.auth-metrics` 保持单列，沿用现有移动端展示逻辑。

在 `560px` 以下继续沿用登录卡片内边距、标题字号和安全说明换行规则，保证输入框和按钮不
溢出。

### 小高度窗口

当窗口高度不足以容纳 hero 和登录卡片时：

- 页面允许自然滚动。
- `.auth-stage` 不使用会强制裁切内容的 `overflow: hidden`。
- 背景装饰仍可固定铺满视口，但内容层必须保持可滚动和可聚焦。

## 数据流与交互

无新增数据流。`RouterView` 仍渲染 `LoginPage.vue`，登录提交、错误提示、密码显隐和成功后
跳转逻辑均保持不变。

## 错误处理与边界

- 背景 canvas 的 resize 逻辑不变，继续跟随 `window.innerWidth` 和 `window.innerHeight`。
- 小屏和小高度场景必须以表单可操作为优先，不因视觉居中裁切输入框、错误提示或登录按钮。
- 装饰层保持 `aria-hidden`，新增内容容器不改变现有可访问性语义。

## 测试与验证计划

- 运行前端类型检查，确认模板和样式调整未引入类型问题：

```text
cd web && npm run typecheck
```

- 使用真实浏览器验证 `/login`：
  - 桌面宽屏下左右内容作为整体居中，不贴两侧。
  - 常见桌面宽度下 hero 与登录卡片垂直中心一致。
  - 平板宽度下切换为单列，登录卡片居中。
  - 手机宽度下表单不溢出，按钮和输入框可完整操作。
  - 小高度窗口下页面可滚动，表单不被裁切。

## 影响范围

- 修改：`web/src/layouts/AuthLayout.vue`。
- 不修改：`web/src/pages/login/LoginPage.vue`、认证 store、路由、后端、OpenAPI 生成产物。

## 不做

- 不重新设计登录卡片视觉。
- 不调整登录字段、placeholder、错误文案或认证流程。
- 不删除背景动画、hero 文案或能力卡片。
- 不新增独立登录页路由或额外布局组件。
