# 文件列表名称列文案设计

## 背景

工作目录、企业知识库、实例知识库三个页面都展示文件或目录列表。目前列表首列统一显示为「名称」。在这些页面里，该列实际表达的是文件或目录的展示名，为了减少和企业名称、实例名称、版本名称等业务实体名称的混淆，需要把这三个文件列表的列头改为「文件名称」。

## 范围

本次只调整以下三个前端页面的表格首列标题：

- `web/src/pages/apps/AppWorkspaceTab.vue`：工作目录文件和目录列表。
- `web/src/pages/knowledge/OrgKnowledgePage.vue`：企业知识库文件列表。
- `web/src/pages/apps/AppKnowledgeTab.vue`：实例知识库文件列表。

## 非范围

- 不修改 API 字段名，`WorkspaceEntry.name` 与 `KnowledgeDocument.name` 继续表示后端返回的展示名。
- 不修改下载、删除、重解析、目录进入、上传等交互逻辑。
- 不修改 OpenAPI、后端接口、数据库字段或生成文件。
- 不全局替换其他页面的「名称」，例如企业名称、实例名称、版本名称、任务名称等保持不变。

## 设计

三个页面保留现有表格结构和数据流，只把文件列表首列的 `title` 从「名称」改为「文件名称」。列 `key` 仍保持 `name`，因为这是组件和后端数据模型的稳定字段，不属于用户可见文案。

工作目录页面中，目录项继续以 `row.name + "/"` 展示并支持点击进入；文件项继续展示 `row.name`。知识库页面中，文件名仍用加粗文本展示，操作列继续基于 document ID 和 `name` 执行下载、删除或重解析。

## 错误处理

本次不改变任何异步请求和错误分支。查询失败、上传超限、下载失败、删除失败、重解析失败等路径沿用现有实现。

## 测试与验证

实现后运行相关前端测试，优先覆盖知识库页面已有组件测试：

- `web/src/pages/knowledge/OrgKnowledgePage.spec.ts`
- `web/src/pages/apps/AppKnowledgeTab.spec.ts`

工作目录当前没有专属组件测试，若没有新增测试，则至少运行前端类型检查或相关测试集合，确认静态文案调整不影响编译。由于这是前端可见文案变更，交付前还需要用真实浏览器查看工作目录、企业知识库、实例知识库三个页面，确认列头均显示为「文件名称」。
