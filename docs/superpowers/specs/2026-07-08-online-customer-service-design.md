# 在线客服子系统 · 需求文档

- 日期：2026-07-08
- 状态：需求评审稿
- 读者：产品经理、业务方

> 本文档描述「在线客服子系统」要做成什么样、给谁用、遵循什么规则，用于需求对齐，不涉及技术实现方案。

---

## 1. 背景与目标

平台现有产品是一套面向企业的 AI 助手管理系统：平台方管理各个企业，每个企业拥有自己的账号体系、账户余额、用量统计和知识库（分行业知识库、企业知识库两类可用内容）。

现在要在这套系统里新增一个「在线客服」子系统，让企业能把 AI 客服投放到自己的官网或落地页上，由 AI 机器人基于企业知识库自动回答访客问题，并在对话中收集销售线索。

### 1.1 我们要解决什么

- 企业希望在自己的网站上提供 7×24 的自动问答，减少人工客服压力。
- 企业希望通过客服对话主动获取潜在客户的联系方式，沉淀销售线索。
- 企业希望了解访客都在问什么、有多少问题没被解决、访客从哪来。

### 1.2 核心目标

1. 企业可创建多个面向不同场景的「客服机器人」，投放到网站或以链接分享，让访客自助问答。
2. 客服机器人基于企业已有的知识库回答问题。
3. 在对话中主动收集访客信息（手机、姓名等，字段可由企业自定义），形成销售线索。
4. 企业可查看完整会话历史与运营统计。
5. 平台方可控制哪些企业能用客服功能、以及每个企业最多能建多少个机器人。
6. 全程有安全约束：访客只能提问，不能借客服做别的事，也不会失控消耗企业费用。

### 1.3 本期不做（后续再议）

- 人工接管 / 转人工坐席（本期纯 AI 应答）。
- 把收集到的线索主动推送到企业的 CRM 或即时通知（本期只做存储、查看、导出）。
- 访客登录 / 访客账号体系（访客始终匿名，一次访问算一个会话）。
- 多语言客服界面。

## 2. 使用角色

| 角色 | 在客服子系统里能做什么 |
|---|---|
| **平台管理员** | 在创建/编辑企业时，决定该企业能否使用客服功能、最多能建几个机器人；本人不参与企业的客服业务 |
| **企业管理员** | **唯一能进入客服子系统的角色**：创建和管理客服机器人、配置知识库与信息收集、查看会话历史、查看统计、导出线索 |
| **企业普通成员** | 没有客服子系统入口，不能进入 |
| **访客** | 企业网站/链接上的匿名用户，只能提问；一次访问算一个会话，无需登录 |

**入口规则**：企业管理员登录后默认进入「概览」页。概览页展示企业状态、常用数据和已开通子系统入口；当企业已被平台开通客服功能时，在概览页的「子系统入口」区域展示「AICC 客服」卡片，点击后进入独立客服工作台。AICC 不放入现有左侧菜单，也不与实例管理页面混在一起。未开通时不显示入口；直接访问客服工作台也会被拒绝。

## 3. 核心概念

- **客服机器人**：企业创建的一个 AI 客服，对应一个业务场景（如售前咨询、售后支持、某产品专线）。一个企业可以有多个机器人，各自独立配置和统计。
- **投放入口**：机器人对外触达访客的方式，分两种——网页挂件、独立链接（详见第 4 节）。
- **会话（Session）**：一位访客的一次完整对话。访客匿名，一次访问就是一个会话。
- **访客信息 / 线索**：在会话中收集到的访客联系方式等信息（手机、姓名等）。
- **知识库**：机器人回答问题所依据的内容来源，来自平台现有的行业知识库、企业知识库，以及可为该机器人单独上传的专属文档。

## 4. 访客怎么进来对话

### 4.1 网页挂件

- 企业管理员为机器人生成一段嵌入代码，贴到自己官网/落地页上。
- 网页右下角会浮出客服气泡，访客点击即展开对话窗。
- 挂件能自动识别访客**从哪个页面进来的（来源页 URL）**和**大致地域**，用于统计。
- 挂件可设置**允许使用的网站域名**，防止嵌入代码被别的网站盗用，白白消耗企业费用。

### 4.2 独立链接 / 二维码

- 每个机器人可生成一个独立的对话页链接，可直接发给客户或做成二维码。
- 适合没有官网的企业。

### 4.3 访客的对话体验

1. 打开后先看到机器人的开场白。
2. 首次对话可展示一句隐私说明（是否展示、文案由企业配置）。
3. 访客自由提问，机器人基于知识库自动回答。
4. 每条回答下方有「有帮助 👍 / 没帮助 👎」按钮，供访客反馈。
5. 机器人会在合适时机自然地邀请访客留下联系方式（手机/姓名等），默认不强制，访客不留也能继续咨询。

### 4.4 一次会话的边界

- 一次访问 = 一个会话；页面刷新可以接着聊，关闭很久后再打开算作新会话。
- 访客不需要登录，也不接入任何用户账号。

### 4.5 功能关闭时

- 如果企业被平台关停客服功能，或机器人被停用、投放入口被下线，网页挂件/链接会显示「客服已下线」，不再应答。

## 5. 企业管理员用的功能

企业管理员进入客服子系统后，有四块功能。

### 5.1 机器人管理

- 查看机器人列表，新建 / 编辑 / 删除 / 启用 / 停用机器人。
- 每个机器人可配置：
  - 基本信息（名称、场景说明）；
  - 开场白；
  - 回答范围（限定只答本企业相关业务）；
  - 关联的知识库（可勾选企业知识库、若干行业知识库，也可为这个机器人单独上传专属文档）；
  - 要收集哪些访客信息（自定义字段，见 5.4）；
  - 投放入口（生成网页挂件代码 / 独立链接，设置允许的网站域名）。
- 新建机器人数量受平台设置的上限约束，达到上限后不能再建，并给出提示。

### 5.2 会话历史

- 按机器人查看所有访客会话列表。
- 点开可看某次会话的完整对话内容、访客留下的信息、来源和地域、是否已解决。
- 支持按时间、是否解决、关键词筛选。

### 5.3 统计看板

展示运营数据，具体指标见第 6 节。

### 5.4 信息收集配置与线索管理

- 企业可为每个机器人**自定义要收集的字段**（字段名称、类型、是否必填、在什么时机问），不限于手机和姓名。
- 收集到的访客信息汇总成销售线索列表，可筛选、导出为 CSV。
- 本期不做主动推送到 CRM 或即时通知。

## 6. 统计指标

| 指标 | 含义 |
|---|---|
| 今日会话数 | 今天新发生的会话数量，可切换时间范围 |
| 会话趋势 | 一段时间内每天/每周的会话量走势 |
| 热门问题 | 访客问得最多的问题及其次数（对相似问题做归类后取前几名） |
| 未解决率 | 没能帮访客解决的会话占比（判定口径见下） |
| 访客地域 | 访客来自哪些省/市的分布 |
| 来源页 URL | 访客从哪些页面进入客服，按页面排名 |
| 留资转化 | 有多少会话最终留下了联系方式 |

**「未解决」如何判定**（综合口径，保证有据可查）：

1. 访客点了「没帮助 👎」或给了差评的会话 → 记为未解决；
2. 没有反馈，但机器人答不上来（触发了「无法回答/建议联系人工」的兜底话术）→ 记为未解决；
3. 其余无明确信号的会话 → 记为「未知」，单独展示，不计入未解决率的分母。

每次会话都会记录「已解决 / 未解决 / 未知」状态，既用于统计，也可在会话历史里按状态筛选。

## 7. 安全与合规要求

访客只能提问，围绕这一点有四类约束：

1. **只答业务相关**：机器人只回答企业业务/知识库范围内的问题，对无关问题（写代码、闲聊、时政等）礼貌拒答并引导回业务。
2. **不能借客服做别的事**：访客只能问答，不能通过客服触发任何操作（下单、改数据、执行其他功能等），也不能套取或篡改机器人的设定。
3. **回答内容安全**：不泄露知识库以外的内部信息，不产生违法有害内容，可配置敏感词过滤。
4. **防刷、防滥用**：对单个访客的提问频率、单次会话的消息数量做限制，可封禁异常访客；配合网站域名限制和企业已有的余额机制，防止客服被恶意刷量、失控消耗费用。

**隐私合规**：首次对话可展示隐私说明，收集到的访客信息安全存储。

## 8. 平台管理员的控制

平台管理员在创建/编辑企业时，新增两个设置：

- **是否开通客服功能**（默认关闭）：
  - 关闭时，该企业管理员看不到、进不去客服子系统；
  - 从开到关时，该企业已经投放出去的挂件/链接立即下线，显示「客服已下线」；
  - 创建企业时可设，之后也能在企业信息里修改。
- **机器人数量上限**（可不限）：限制该企业最多能创建几个客服机器人。

企业使用客服产生的 AI 费用，计入企业现有的账户余额，不单独计费。

---

## 9. 与现有系统的关系

客服子系统是现有平台的一个新模块，复用现有能力：

- **共用账号体系、账户余额、用量统计**：客服消耗计入企业现有余额，用量并入现有统计。
- **复用知识库**：机器人直接使用平台已有的行业知识库、企业知识库，并可额外上传专属文档。
- **沿用企业管理员权限**：只有企业管理员能进入，与现有权限体系一致。
- **进入方式**：企业管理员从「概览」页的「子系统入口」区域进入 AICC；点击后跳转到独立客服工作台，工作台拥有自己的导航和页面布局，不复用现有实例管理页面。

## 10. 待产品确认的开放问题

以下是需求评审时建议和产品经理进一步拍板的点。每条给出当前**暂定方案**，便于讨论时直接确认或调整。

| # | 问题 | 考量 | 暂定方案 |
|---|---|---|---|
| 1 | **挂件外观能否定制品牌** | 企业是否希望客服窗口用自己的颜色、logo、客服名称、头像，保持品牌一致 | 本期支持基础定制（客服名称、头像、主题色）；更复杂的样式后续再议 |
| 2 | **访客能否发图片/文件** | 售后场景访客常需发故障截图；纯文字会限制体验 | 本期只做文字问答，图片/文件上传后续再议 |
| 3 | **必填字段是否阻断对话** | 若某收集字段设为必填，是"不填就不能继续问"还是"仍可继续、只是记为未留资" | 暂定：允许设为必填，必填时访客不提供则无法继续；建议默认都设为非必填 |
| 4 | **隐私同意的强度** | 是否必须访客明确同意才能对话/才能收集信息（涉及个人信息保护合规） | 暂定：展示隐私说明即可，不强制勾选同意；如合规要求更严再加"同意才能继续" |
| 5 | **数据保留期限** | 会话记录和访客线索保存多久，涉及隐私合规与存储成本 | 暂定：长期保留、可手动删除；是否设自动过期请产品确认 |
| 6 | **新线索是否需要即时提醒** | 虽不做完整 CRM 对接，但企业可能希望有新线索时收到通知（站内/邮件等） | 暂定：本期仅在后台可查看和导出，不做即时提醒；若价值高可作为优先补充项 |
| 7 | **未解决率分母口径** | "未知"状态的会话是否计入分母，直接影响未解决率数值大小 | 暂定：未知不计入分母、单独展示；请产品确认此口径 |
| 8 | **是否要防失控消费的硬上限** | 除限流外，是否需要给企业客服设"每日会话/消息上限"防止余额被快速消耗 | 暂定：只靠限流 + 企业余额自然兜底，不设硬上限；若担心失控可加 |
| 9 | **回头客识别** | 同一访客多次来访，是否关联识别为同一人（利于线索去重和跟进，但涉及隐私） | 暂定：不跨会话关联，每次访问独立；如需回头客识别再议 |
| 10 | **答不上来时的引导** | 机器人无法回答时，除标记未解决外，是否引导访客留联系方式以便后续跟进 | 暂定：无法回答时引导留资（复用信息收集能力）；文案可配置 |
| 11 | **会话超时时长** | 访客关闭页面多久后再打开算作新会话 | 暂定：设一个默认超时时长（如 30 分钟），可后续调整 |
| 12 | **是否支持多语言提问** | 访客用其他语言提问时机器人是否作答 | 暂定：按知识库和模型能力自然应答，不单独做多语言界面 |

## 11. 需求确认清单

- [x] 访客入口：网页挂件 + 独立链接
- [x] 客服大脑：企业可建多个客服机器人，各对应一个场景，独立配置和统计
- [x] 安全限制：只答业务 + 不能操作/篡改 + 内容安全 + 防刷防滥用，四类全覆盖
- [x] 信息收集：企业自定义字段，默认不强制，支持查看与导出 CSV，不做 CRM 推送
- [x] 未解决率：综合口径（反馈优先，答不上来兜底），会话记录三态状态
- [x] 知识库：企业库 + 行业库（可勾选）+ 机器人专属文档
- [x] 平台控制：是否开通客服 + 机器人数量上限；关闭时挂件下线
- [x] 子系统仅企业管理员可进入
