# 在线客服子系统 · 需求设计文档

- 日期：2026-07-08
- 状态：需求评审稿（仅需求，不含实现细节）
- 目标读者：产品、平台研发、企业侧对接方

> 本文档用于与产品对齐需求边界，定义「做什么、边界在哪、口径怎么算」，不涉及具体技术实现方案。实现拆分（数据库 migration、API 契约、前端组件）在需求定稿后另出实现计划。

---

## 1. 背景与目标

现有平台是多租户 AI 助手管理系统：平台管理员管理企业（organization），每个企业有 AI 实例、三层知识库（行业库 / 企业库 / 实例库）、共享账号体系与 new-api 余额/用量。

现有「对话功能」面向**企业内部人员登录后台查看会话**。本次要新增的「在线客服子系统」本质不同：面向**企业官网/落地页的外部匿名访客**，由 AI 机器人基于企业知识库自动应答，并在对话中收集销售线索。

### 1.1 核心目标

1. 企业可创建多个面向不同场景的「客服机器人」，投放到自己的网站或以链接分享，让匿名访客自助问答。
2. 客服机器人复用企业已有的知识库能力，回答业务问题。
3. 在对话中主动收集访客信息（手机、姓名等，字段可自定义），沉淀为销售线索。
4. 企业管理员可查看完整会话历史与运营统计。
5. 平台管理员控制哪些企业可用客服功能、以及机器人数量上限。
6. 全程有安全约束：访客只能提问，不能触发任何操作类能力，且不失控烧钱。

### 1.2 非目标（本期不做，YAGNI）

- 人工接管 / 转人工坐席。
- 收集线索的主动推送 / CRM 系统对接（本期只做存储、查看、导出 CSV）。
- 访客侧登录 / 接入用户体系（访客始终匿名，一次访问一个 session）。
- 多语言客服界面。
- 独立于余额之外的会话/消息计费与配额。

## 2. 角色与权限边界

| 角色 | 在客服子系统中的能力 |
|---|---|
| **平台管理员** | 创建/编辑企业时设置「是否开通客服」总开关与「机器人数量上限」；**本人不进入企业客服业务界面** |
| **企业管理员**（org_admin） | **唯一可进入客服子系统的角色**：创建/管理客服机器人、配置知识库与信息收集、查看会话历史、查看统计、导出线索 |
| **企业成员**（org_member） | 无客服子系统入口，不可进入 |
| **匿名访客** | 通过挂件/链接进入；纯问答；一次访问 = 一个 session；不登录、不接入用户体系 |

**入口要求**：企业管理员首页（`/org-console`）新增「在线客服」入口（卡片 + 导航项）。仅当该企业已被平台开通客服功能时入口可见且可进入；未开通时入口隐藏或显示「未开通」提示。

**新增鉴权谓词**（统一放 `internal/auth/authorizer.go`，不在 handler/service 内联判断角色）：

- `CanAccessCustomerService(principal, orgID)`：org_admin 且该企业 `customer_service_enabled = true`。
- `CanManageCSBot` / `CanViewCSSessions` / `CanExportCSLeads` 等：均在上一条基础上限定本企业范围；平台管理员按现有惯例可跨企业只读（如需）。

## 3. 核心概念与数据实体

设计原则：**对话消息正文复用底层对话引擎存储，不重复落库；客服专属的结构化数据（机器人配置、访客会话元数据、访客画像、反馈）由本子系统独立建表存储。** 本子系统通过关联 id 回引擎拉取消息历史。

> 下列为需求层面的实体与关键字段，最终库表结构、字段类型、索引在实现计划中确定。所有新表须带表与字段的中文 SQL COMMENT。

### 3.1 客服机器人 `cs_bots`

企业下可创建多个，各对应一个业务场景（如售前咨询、售后支持、某产品专线）。

- 所属企业 id
- 名称、场景描述
- 状态：启用 / 停用
- 开场白文案
- 话题边界配置（业务范围描述，用于约束 AI 应答范围）
- 绑定的底层对话引擎实例引用（对企业管理员屏蔽，属实现细节）
- 检索知识库范围：是否含企业知识库、勾选的行业知识库 id 列表、该机器人专属文档库引用
- 限流参数（见 §7.4）
- 隐私声明文案（可选，见 §5.4）
- 创建时间、更新时间

### 3.2 投放入口 `cs_channels`

每个机器人可有一个或多个投放入口。

- 所属机器人 id
- 类型：网页挂件 / 独立链接
- 唯一投放 token
- 域名白名单（挂件类型用于防盗刷，见 §7.4）
- 上线状态：上线 / 下线
- 创建时间

### 3.3 访客会话 `cs_sessions`

一次访客对话。

- 所属机器人 id、投放入口 id
- session token（浏览器侧持有，用于续接）
- 访客匿名标识（浏览器指纹或匿名 id）、来访 IP
- 来源页 URL、referrer
- 地域（由 IP 解析到省/市）
- 设备 / User-Agent
- 开始时间、最后活跃时间
- 解决状态：`resolved` / `unresolved` / `unknown`
- 消息数
- 关联的底层引擎会话 id（用于拉取消息正文）

### 3.4 访客画像 `cs_visitor_profiles`

一次会话收集到的访客信息（键值结构，关联 session）。

- 所属 session id
- 字段 key、字段值
- 各字段收集状态（已收集 / 未收集）

### 3.5 信息收集字段配置 `cs_collect_fields`

每个机器人自定义要收集的字段。

- 所属机器人 id
- 字段名（key）、展示标签、字段类型（文本/手机/邮箱等）
- 是否必填
- 收集时机（开场即问 / 对话中择机 / 结束前）
- 排序

### 3.6 消息级反馈 `cs_message_feedback`

- 关联 session id、消息标识
- 反馈类型：👍 有帮助 / 👎 没帮助
- AI 兜底命中标记（该条回答是否触发了「无法回答/建议人工」兜底）
- 知识库是否命中标记
- 时间

## 4. 访客端（挂件 + 独立链接）

### 4.1 网页挂件（embed widget）

- 企业管理员为机器人生成一段 JS 嵌入代码，贴入自己官网/落地页。
- 页面右下角浮出客服气泡，点击展开对话窗。
- 挂件自动上报**来源页 URL、referrer、User-Agent**；后端按来访 IP 解析**地域**——满足统计维度（§6）。
- 挂件加载时校验**域名白名单**：仅允许配置的域名加载该 token，防止 token 被盗用到第三方站点消耗企业余额。

### 4.2 独立链接 / 二维码

- 每个机器人可生成独立对话页 URL（含唯一 token），可直接分享或转二维码。
- 覆盖没有官网的企业。

### 4.3 访客对话体验

1. 打开即显示机器人开场白。
2.（可选）首次对话展示隐私声明/同意提示（文案可配置）。
3. 访客自由提问，AI 基于绑定知识库检索作答。
4. 每条回答下带 👍/👎 反馈按钮。
5. AI 按配置时机自然引导访客留资（手机/姓名等自定义字段）；默认非强制，访客不给也能继续（除非某字段被设为必填才能继续）。

### 4.4 会话生命周期

- 一次访客访问 = 一个 session。
- 浏览器侧保留 session token，页面刷新可续接同一会话；关闭后超时再进则视为新 session。
- 不做访客登录、不接入用户体系。

### 4.5 功能关闭时的表现

- 企业被平台关停客服功能，或机器人被停用、投放入口被下线时：挂件/链接显示「客服已下线」，停止应答、不再消耗余额。

## 5. 企业管理端（子系统主界面）

企业管理员进入客服子系统后有四块功能。

### 5.1 机器人管理

- 机器人列表 + 新建/编辑/删除/启停。
- 配置项：基本信息（名称、场景描述）、开场白、话题边界、绑定知识库（勾选企业库 / 行业库 + 为该机器人上传专属文档）、信息收集字段（自定义增删）、限流参数、投放入口（生成挂件代码 / 独立链接、设置域名白名单）、启用/停用。
- 新建机器人受平台设置的「机器人数量上限」约束：达上限禁止新建并提示。

### 5.2 会话历史

- 按机器人筛选，列出该机器人所有访客会话。
- 可查看单次会话完整逐条消息、访客画像、来源/地域、解决状态。
- 支持按时间范围、解决状态、关键词筛选。

### 5.3 统计看板

见 §6 指标口径。

### 5.4 线索管理

- 汇总所有会话收集到的访客信息（画像）。
- 支持筛选与导出 CSV。
- 本期不做 CRM 推送 / 主动通知。

### 5.5 隐私与合规

- 首次对话可展示隐私声明/同意提示，文案由企业管理员配置。
- 收集到的访客信息加密存储。

## 6. 统计口径（明确定义）

| 指标 | 口径定义 |
|---|---|
| 今日会话数 | 当日新建 session 数；支持切换时间范围 |
| 会话趋势 | 按日/周会话量折线 |
| 热门问题 | 对访客提问做归类聚合（语义或关键词聚类），取 Top N 高频问题及其次数。**具体聚合方式在实现计划中定，需求层面只要求「展示高频问题及次数」** |
| 未解决率 | 组合口径：① 有 👎/差评的会话 → 未解决；② 无反馈但 AI 触发兜底话术或知识库未命中 → 未解决；③ 其余为 `unknown`。**分母口径**：默认按「有明确解决/未解决信号的会话」计（unknown 单列展示，不进分母）；此口径在文档中固定，实现须一致 |
| 访客地域 | 按会话来访 IP 解析到省/市，做分布 |
| 来源页 URL | 挂件上报的来源页，按 URL 聚合取 Top |
| 留资转化 | 完成留资（至少收集到一个字段）的会话数 / 总会话数 |

每条会话在产生时即落 `resolved / unresolved / unknown` 解决状态，保证「未解决率」有据可查，也支持会话历史按状态筛选。

## 7. 安全限制

覆盖四类，全部为本期需求。

### 7.1 话题边界

- 机器人可配置业务范围。AI 对与业务无关的问题（写代码、闲聊、时政等）礼貌拒答并引导回业务，通过系统提示词约束。

### 7.2 防注入 / 防越权

- 纯问答：访客无法触发任何操作类能力（工具调用、技能执行、下单、改数据等一律不对访客开放）。
- 防 prompt injection 套取系统提示词、防访客冒充管理员或篡改机器人行为。

### 7.3 输出安全

- 不泄露绑定知识库之外的内部信息。
- 不生成违法有害内容。
- 支持配置敏感词/黑名单。

### 7.4 频率与滥用限制（防刷余额）

- 单访客 / 单 IP 限流。
- 单会话消息数上限。
- 异常访客封禁能力。
- 挂件域名白名单校验（见 §4.1），防 token 被第三方站点盗用消耗余额。
- 烧钱风险由「访客限流 + 企业现有 new-api 余额体系」自然兜底，不单设会话/消息级配额。

## 8. 平台管理端

在现有「创建 / 编辑企业」表单中新增两个字段（对齐现有 `max_instance_count`、知识库配额的建模方式）：

- **客服功能开关** `customer_service_enabled`（布尔，默认关闭）：
  - 关闭时企业管理员看不到 / 进不去客服子系统。
  - 从开到关时，该企业已投放的挂件 / 链接立即下线，显示「客服已下线」。
  - 支持创建时设置，也支持后续在企业详情中修改。
- **机器人数量上限** `cs_bot_limit`（整数，NULL = 不限）：限制该企业最多可创建的机器人数量，达上限禁止新建。

## 9. 与现有系统的复用关系

| 现有能力 | 复用方式 |
|---|---|
| 底层对话引擎（hermes / oc-ops） | 作为机器人对话「大脑」，承载消息正文存储与检索式应答 |
| 三层知识库（行业 / 企业 / 实例）+ RAGFlow 检索 | 机器人绑定知识库、专属文档库；应答时合并召回 |
| new-api 余额 / 用量代理 | 客服消耗计入企业余额；不新增本地缓存或独立计费 |
| org_admin 鉴权体系 | 新增 `Can*` 谓词，统一放 `authorizer.go` |
| audit_logs 审计 | 客服关键操作（建/删机器人、导出线索、启停等）落审计 |
| 前端 DashboardLayout / 路由框架 | 客服子系统作为企业管理员导航下的新模块 |

## 10. 待实现计划阶段细化的开放点

以下为需求已定、但实现方式待计划阶段决策的点（不影响需求评审）：

- 「热门问题」的具体聚合算法（语义聚类 vs 关键词）。
- 「底层每个机器人映射到专属引擎实例」的具体承载方式（一 bot 一实例 vs 共享实例多配置）。
- IP 地域解析所用数据源。
- 挂件的前端打包与跨域投放方案。

## 11. 需求确认清单（评审对齐用）

- [x] 访客入口：网页挂件 + 独立链接（共用后端）
- [x] 客服大脑：企业可建多个客服机器人，底层复用专属对话引擎，对企业管理员屏蔽实例概念
- [x] 安全限制：话题边界 + 防注入防越权 + 输出安全 + 限流防刷，四类全覆盖
- [x] 信息收集：企业自定义字段，默认非强制，存储 + 查看 + 导出 CSV，不做 CRM 推送
- [x] 未解决率：组合口径（反馈优先，兜底看 AI），会话落三态
- [x] 知识库：企业库 + 行业库（勾选）+ 机器人专属文档，合并召回
- [x] 平台控制：总开关 + 机器人数量上限；关闭时挂件下线
- [x] 子系统仅企业管理员可进入
