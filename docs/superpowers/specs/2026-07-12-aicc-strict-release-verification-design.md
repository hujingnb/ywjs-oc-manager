# AICC 严格上线验证设计

- 日期：2026-07-12
- 状态：已确认，等待规格审阅
- 验证环境：本地 k3d（允许清空并重建验收数据）

## 1. 目标与结论边界

本验证判断 AICC 在线客服子系统是否满足直接上线门禁。结论只接受本轮生成、可复核的证据：自动化结果、真实 Chrome 浏览器行为、环境日志和测试报告。

本地 k3d 结果只说明当前代码在该同构程度有限的环境中通过门禁，不代替生产节点规格、网络、存储和外部服务配额的验收。任何关键项未执行、环境受限或失败，最终均为 `NO-GO`，不会降低门槛。

只有覆盖矩阵没有影响上线判断的 `FAIL` 或 `BLOCKED`，且后续各层全部通过，才能给出 `GO`。

## 2. 分层门禁

按顺序执行；每一层通过后进入下一层。发现失败时保留证据、补充可复现自动化用例、进行最小修复，并重新运行当前层及受影响的上游回归。

1. 基础门禁：Go 全仓测试、前端 Vitest、TypeScript 类型检查、前端生产构建、OpenAPI 同步检查和全部 AICC Playwright 用例。
2. 需求补测：逐条覆盖既有矩阵中的未验证项，包括智能体管理与上限、投放链接/二维码/挂件来源与白名单、会话来源地域详情、留资规则、知识库失败路径、上传限制、提示词注入、频率与数量限制、封禁/敏感词/余额不足、国际化、移动工作台、统计口径、GeoIP 和数据保留。
3. 真实浏览器：在 Chrome 中覆盖 platform_admin、org_admin、org_member 和匿名访客；对每个场景同时核验可见页面、网络响应、控制台错误和服务日志。
4. 故障恢复：在本地 k3d 分别停止并恢复 Hermes、RAGFlow、new-api、Redis、MySQL 和 manager-api。验证错误提示、超时和幂等边界、恢复后续聊、不会重复创建会话/消息/线索，以及恢复后的核心浏览器冒烟。
5. 容量与变更安全：执行 100 并发、持续 30 分钟的访客压测；成功率至少 99.5%，完整消息 P95 不超过 15 秒，且无异常重启或跨租户串写。随后执行 `master -> 当前版本 -> 回滚 -> 恢复当前版本` 的迁移、健康检查和核心业务验证。

## 3. 覆盖完整性

覆盖范围以“需求矩阵 + 代码反查”双向校验，不能只依赖已有测试名称或历史结果。

- 需求侧逐项覆盖企业开通与关闭、智能体管理、投放入口、会话、留资与线索、三类知识库、图片消息、安全限制、权限、国际化、移动端、统计、GeoIP、数据保留、故障恢复、容量、升级和回滚。
- 代码侧反查当前分支相对 `master` 的全部 AICC 相关变更，包括路由、API、service、权限谓词、SQL、迁移、定时任务、前端页面、公开挂件、Hermes runtime 和部署配置。任何没有需求矩阵条目的行为都必须补入矩阵。
- 每项功能至少验证正常路径、关键异常、边界值、权限边界和故障恢复；不适用的维度必须记录明确依据，不能以缺少用例默认视为通过。
- 既有 `PASS` 仅代表历史证据。最终结论必须在同一最终提交、同一组镜像和重建后的干净数据上重新取得证据。

## 4. 缺陷闭环

发现问题后保存输入、截图、网络摘要、浏览器控制台错误、相关服务日志和复现步骤。能够自动化复现的问题先补失败用例，再做根因级最小修复。

修复后依次运行针对性测试、受影响模块测试和此前已通过的上游门禁。所有缺陷处理完成后，必须清空验收数据并从最终提交完整复跑；不得拼接不同提交或不同环境的零散结果形成通过结论。

如果本地资源或外部依赖导致关键项无法执行，该项保持 `BLOCKED`，最终结论为 `NO-GO`，不得降低门槛。

## 5. 证据与交付

- 每层记录命令、开始和结束时间、Git 基线、镜像 digest、迁移版本、用例结果与失败分类。报告不包含本地账号、令牌或访客隐私内容。
- 浏览器场景保留必要的截图、网络摘要、控制台错误和关联服务日志；压测记录成功率、P50/P95/P99、错误分类、资源曲线和重启情况。
- `docs/testing/aicc-requirement-matrix.md` 仅按本轮证据更新为 `PASS`、`FAIL` 或 `BLOCKED`；既有历史状态不能作为本轮通过依据。
- 完成后清理临时故障配置、压测数据和调试产物，恢复本地环境。修复按业务边界以中文 Conventional Commits 单独提交。
- 最终报告明确给出 `GO` 或 `NO-GO`。`NO-GO` 必须列出每个阻塞项的复现步骤、证据位置和上线风险。

## 6. 成功判定

通过需要同时满足：自动化和真实浏览器全部通过；四类身份权限边界完整；没有 P0/P1、跨租户、消息丢失、重复扣费或数据不一致；故障恢复、容量、升级和回滚均达标；最终镜像、数据库版本、配置和测试报告能追溯到同一提交；工作区不混入密钥、临时数据或未提交修复。

本规格范围只限本地验证和必要的测试/缺陷修复；不访问或变更生产集群。
