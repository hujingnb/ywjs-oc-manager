# AICC 非容量生产就绪验证设计

- 日期：2026-07-13
- 状态：已确认
- 范围：AICC 在线客服的全部非容量上线门禁
- 明确排除：100 并发、30 分钟、P95 `<= 15s` 容量门禁；该项保留为已知生产风险，不得表述为已达标

## 1. 目标和结论边界

本验证消除现有生产报告“部分场景已通过”与需求覆盖矩阵大量 `BLOCKED` 的证据不一致。以当前 `kefu` 分支的单一提交、单一镜像和干净本地 k3d 数据环境为基线，逐项验证所有非容量 AICC 需求，并产出可复核证据。

最终结论只能是：

- `PASS`：本轮在固定基线执行成功，保留命令、时间、提交、镜像和结果证据。
- `FAIL`：行为不符合需求；先建立可复现测试，再修复并回归。
- `BLOCKED`：环境或外部条件无法执行；本轮不允许遗留。
- `N/A`：仅用于容量门禁，记录其排除依据和既有 P95 风险。

本轮不能以历史零散测试、代码阅读或相似场景替代本轮证据，也不能因排除容量门禁而宣称完整生产 `GO`。

## 2. 验证架构

所有覆盖矩阵条目必须映射到至少一种以下验证方式，并在同一基线运行：

1. 代码级：Go、Hermes、前端单元测试，TypeScript 类型检查，前端构建与 OpenAPI 契约检查。
2. API 与数据级：固定种子账号和隔离企业下的写入、查询、权限和跨租户拒绝验证。
3. 真实浏览器：Chrome 覆盖平台管理员、企业管理员、企业成员和匿名访客；覆盖桌面及需要响应式确认的移动视口。
4. 环境级：最终镜像内容与健康状态、GeoIP 内置库和更新、留存调度、依赖故障恢复、升级与回滚。

每项证据记录需求 ID、执行命令或步骤、结果、提交、镜像标签、时间和必要的截图、网络请求或日志。证据不记录真实令牌、密码或访客隐私数据。

## 3. 分层执行

按顺序执行；任一层出现失败，完成缺陷闭环并重新验证受影响层后才进入下一层。

1. 固化基线：确认当前提交、工作区、镜像标签、迁移版本和验收种子数据；清理无关测试数据。
2. 静态门禁：运行全仓测试、类型检查、生产构建、OpenAPI 检查、镜像构建和敏感信息检查。
3. 业务矩阵：验证平台配置、智能体、公开链接/二维码/挂件、会话、线索、知识库、统计、GeoIP 和留存。
4. 权限与安全：验证四类身份、跨企业访问、伪造令牌、域名白名单、限流、封禁、敏感词、上传校验和提示词注入边界。
5. 故障恢复：分别注入 Hermes、RAGFlow、new-api、Redis、MySQL 和 manager-api 故障，确认错误提示、恢复续聊、幂等与数据一致性。
6. 升级回退：从 `master` 基线升级到当前版本，执行受控回退，再恢复当前版本并复跑核心浏览器流程。
7. 汇总：更新需求覆盖矩阵、证据目录和生产就绪报告，将容量项记为 `N/A` 并保留既有 P95 风险。

## 4. 业务覆盖

覆盖矩阵中的全部非容量需求必须完成，包括：

- 企业 AICC 开通、数量上限、智能体完整生命周期和跨模块智能体切换。
- 公开链接、二维码、挂件来源和域名白名单。
- 延迟创建 session、恢复、新建对话、零消息过滤、筛选、分页、来源、地域与详情。
- 三态会话、留资字段、恢复会话、线索、关联对话和 CSV 导出。
- 当前客服库、企业库、授权行业库的上传、解析、检索、撤销、下载、删除、无匹配和 RAGFlow 故障路径。
- 图片、消息限额、频率限制、访客封禁、敏感词、余额不足及提示词注入。
- 中英文完整用户可见文案、工作台及公开页的桌面/移动可操作性。
- 趋势、地域、来源、问题、线索和未解决率统计口径。
- IPv4/IPv6 GeoIP 解析、国内更新源有效更新和留存清理。

## 5. 缺陷闭环和通过条件

每个失败都保存复现步骤、输入、截图、网络请求和服务日志；可自动化的失败先加入测试，再修改实现。修复按业务阶段提交，随后重跑当前层和受影响的上游回归。

完成时必须同时满足：

- 除容量门禁外，覆盖矩阵无 `FAIL` 或 `BLOCKED`。
- 自动化、API、真实浏览器、故障恢复和升级回退证据均指向同一最终提交和镜像。
- 不存在 P0/P1 缺陷、跨租户访问、消息丢失、重复创建会话或数据不一致。
- 工作区干净，测试产物不含密钥或隐私数据。

符合时的结论限定为“非容量功能门禁通过；100 并发 P95 风险未关闭，不构成完整生产 `GO`”。
