# AICC 未完成项补齐设计

- 日期：2026-07-10
- 状态：设计确认稿
- 关联需求：`docs/superpowers/specs/2026-07-08-online-customer-service-design.md`

## 1. 背景

在线客服子系统的主体能力已经落地：企业可创建客服机器人、配置知识范围、投放公开入口、接收访客消息、沉淀会话和线索。原需求文档中仍有若干明确承诺项尚未闭环，主要集中在投放配置、会话续接、运营统计和安全治理。

本设计用于补齐这些明确承诺项，不扩展到语音客服、人工坐席、CRM 推送或更复杂的运营自动化。

## 2. 名词约定

后续代码、目录、接口、表名和前端模块继续使用 `aicc` 作为业务标识。

`AICC` 的含义统一为 **AI Integrated Customer Care**，表示统一承载在线客服、知识问答、留资、安全治理和未来语音客服接入的智能客服子系统。

## 3. 本轮范围

本轮只补齐原需求文档中已经明确写下的未完成内容：

- 独立链接支持二维码展示和下载。
- 网页挂件和独立链接记录访客大致地域，并在后台展示和统计。
- 页面刷新后优先恢复同一会话，关闭较久或 token 过期后再创建新会话。
- 会话历史支持按时间、地域、解决状态筛选。
- 统计看板支持时间范围切换、日/周会话趋势、未解决率、访客地域分布、来源页排名和高频问题聚合。
- 公开端支持敏感词过滤、单会话消息数上限和异常访客封禁。

不在本轮做：

- 语音客服、通话记录、坐席接管。
- CRM 或即时通知推送。
- 复杂语义聚类、统计快照仓储、BI 报表系统。
- 跨会话回头客识别。

## 4. 方案选择

采用“运营配置中心化”方案：能力仍沿用现有 `aicc_*` 模块扩展，但把安全策略、续接策略和统计口径放入统一配置边界，避免新增能力散落在公开端 service、后台 handler 和前端页面中。

这样可以保持当前 AICC 主体结构稳定，也为后续语音客服复用同一套运营配置和安全治理边界留出空间。

## 5. 模块划分

### 5.1 投放配置

投放配置负责独立链接、二维码、站点白名单和会话续接。

- 独立访问链接继续由公开 token 生成。
- 二维码由前端根据独立链接实时生成，支持下载 PNG，不在数据库存储图片。
- 访客端使用 localStorage 保存最近 session token。
- 页面刷新或重新打开时先校验 session token，未过期且归属当前 agent 时恢复原会话。
- token 无效、过期、agent 停用或企业客服功能关闭时，不恢复旧会话。

### 5.2 会话运营

会话运营负责后台筛选和会话详情展示。

- 会话列表增加时间范围、地域、解决状态筛选。
- 会话摘要展示地域、来源页、消息数、最近消息时间和解决状态。
- 会话详情展示来源页、referrer、地域、访客消息、助手消息、反馈和线索字段值。
- 地域为空时显示为未知，不影响筛选和详情加载。

### 5.3 数据分析

数据分析负责后台总览页。

- 统计接口支持 `start_at`、`end_at`、`bucket=day|week`。
- 默认时间范围为近 7 天。
- 单次查询最大范围限制为 180 天，避免后台聚合查询过重。
- 返回核心指标、会话趋势、未解决率、访客地域分布、来源页排名、高频问题。
- 高频问题本轮先做文本规范化聚合：去首尾空白、合并连续空白、统一大小写和常见标点，不做语义相似聚类。
- 未解决率沿用原需求口径：明确未解决会话除以已知状态会话，未知状态单独展示，不计入分母。

### 5.4 安全治理

安全治理负责公开端消息发送前的统一拦截。

- 敏感词命中时拒绝发送，并返回统一错误码。
- 单会话消息数超过上限时拒绝继续发送。
- 异常访客封禁命中时拒绝发送。
- 被拒绝的消息不调用 Hermes，不消耗模型费用。
- 前端展示温和提示，不泄露敏感词规则、封禁阈值或内部策略细节。

## 6. 数据模型

### 6.1 `aicc_agent_settings`

新增按机器人维度的运营配置表。

字段为：

- `agent_id`：机器人 ID，唯一。
- `message_limit_per_session`：单会话消息数上限。
- `sensitive_words_json`：敏感词数组 JSON。
- `blocked_visitor_enabled`：是否启用异常访客封禁。
- `blocked_visitor_threshold_json`：封禁阈值配置 JSON。
- `session_resume_ttl_minutes`：刷新续接有效期。
- `analytics_config_json`：统计口径预留配置。
- `created_at`、`updated_at`。

默认设置由 service 层补齐，旧机器人没有配置记录时也能按默认策略运行。

### 6.2 `aicc_blocked_visitors`

新增被封禁访客表。

字段为：

- `id`：封禁记录 ID。
- `agent_id`、`org_id`：归属机器人和企业。
- `visitor_hash`：访客哈希，使用现有 IP / UA 哈希能力组合生成，不保存明文 IP。
- `reason`：封禁原因。
- `expires_at`：封禁到期时间。
- `created_at`、`updated_at`。

封禁到期后不再拦截。解除封禁可以通过后台设置页操作。

### 6.3 复用现有表

- `aicc_sessions` 继续保存 `region`、`source_url`、`referrer`、`ip_hash`、`user_agent_hash`、`resolved_status`、`expires_at`。
- `aicc_messages` 继续作为会话趋势、高频问题和消息数上限的数据来源。
- `aicc_feedback` 继续作为未解决率判定来源之一。

本轮不新增统计快照表。若后续查询性能不足，再按日报表或物化统计单独设计。

## 7. 后端接口

### 7.1 管理端设置接口

新增：

- `GET /api/aicc/agents/{id}/settings`
- `PUT /api/aicc/agents/{id}/settings`

权限继续放在 `internal/auth/authorizer.go` 的 AICC 权限判断中，service 不新增本地 `canX` 函数。

### 7.2 管理端统计接口

扩展：

- `GET /api/aicc/agents/{id}/analytics?start_at=&end_at=&bucket=day|week`

返回：

- 指标卡：会话数、未解决率、未知会话数、留资转化。
- 趋势：按日或周分桶的会话数。
- 分布：访客地域、来源页 URL。
- 排名：高频问题。

无数据时返回 0 和空数组，不把空数据当错误。

### 7.3 管理端会话接口

扩展：

- `GET /api/aicc/agents/{id}/sessions?start_at=&end_at=&region=&resolved=`

筛选参数缺省时保持现有列表行为。

### 7.4 公开端会话接口

公开端创建或恢复会话时：

1. 读取访客端传入的 session token。
2. 校验 token 是否存在、未过期、归属当前 agent。
3. 成功时返回原会话和历史消息。
4. 失败时创建新会话。

创建新会话时尝试解析粗粒度地域。地域解析失败只记录日志，不阻塞会话创建。

### 7.5 公开端消息发送流程

消息发送前统一执行：

1. agent 和企业客服状态校验。
2. 封禁访客校验。
3. 单会话消息数上限校验。
4. 敏感词过滤。
5. 隐私和留资校验。
6. 调用 Hermes。
7. 保存访客消息和助手消息。

前置校验失败时不调用 Hermes。

## 8. 前端设计

### 8.1 总览页

总览页顶部保留核心指标卡，增加时间范围切换和 `day/week` 粒度切换。

页面展示：

- 会话趋势图。
- 未解决率和未知状态数量。
- 访客地域分布。
- 来源页排名。
- 高频问题。

筛选条件同步到 URL query，刷新页面后保持当前统计范围。

### 8.2 会话页

会话列表增加：

- 时间范围筛选。
- 地域筛选。
- 解决状态筛选。

列表列增加：

- 地域。
- 来源页。
- 消息数。
- 最近消息时间。

会话详情保留现有聊天记录展示，并补充来源和地域信息。

### 8.3 设置页

设置页新增“运营配置”区域：

- 独立链接复制。
- 二维码展示和下载。
- 会话续接开关和有效期。
- 单会话消息数上限。
- 敏感词配置。
- 异常访客封禁策略。
- 当前封禁访客列表和解除操作。

页面保持后台工具型信息密度，不新增营销式独立页面。

### 8.4 公开客服页和 widget

访客端：

- 保存并复用 session token。
- 刷新后自动恢复原会话。
- 恢复失败时自动创建新会话。
- 敏感词、超限、封禁错误统一展示为用户可理解的提示。

阶段拆分为：

1. `feat(aicc): 增加运营配置数据模型`

## 11. 验收标准

- 后台设置页能复制独立链接、展示并下载二维码。
- 访客刷新页面后能继续同一会话；token 过期后创建新会话。
- 新会话能记录粗粒度地域，后台列表、详情和统计中可见。
- 会话列表能按时间、地域、解决状态筛选。
- 总览页能按时间范围和日/周粒度展示趋势、未解决率、地域分布、来源页和高频问题。
- 敏感词、单会话消息数上限、异常访客封禁均能阻止公开端继续消耗模型调用。
- 管理端权限仍只允许平台管理员和企业管理员访问，企业普通成员无入口和接口权限。
- `go test ./...`、前端类型检查、前端测试、OpenAPI 校验和真实浏览器验证通过，或在交付说明中明确不能运行的原因和风险。
