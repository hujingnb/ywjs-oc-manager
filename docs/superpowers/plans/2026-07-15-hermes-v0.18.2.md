# Hermes v0.18.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 新增以语义版本号标识的 Hermes v0.18.2 runtime 变体，将其设为普通实例默认版本，同时保持历史日期版本可构建、可回退。

**Architecture:** 普通 runtime 继续以 `version.txt` 作为本项目变体和镜像版本的唯一来源，并新增可选 `hermes-ref.txt` 表达上游 Git ref；Makefile 对新变体读取显式 ref，对历史变体回退到原版本值。新变体整体继承 v0.18.0 harness，配置列表新增 v0.18.2 且不删除旧项。

**Tech Stack:** GNU Make、Docker build arguments、YAML、Python 3.13/pytest、POSIX shell

---

## 文件结构

- `Makefile`：解析普通 Hermes runtime 的产品版本和上游源码 ref，并把两者分别用于镜像 tag 与 Docker 构建参数。
- `runtime/hermes/hermes-v0.18.2/`：从 `hermes-v2026.7.1` 复制的自包含 runtime build context；`version.txt` 保存 `v0.18.2`，`hermes-ref.txt` 保存 `v2026.7.7.2`。
- `deploy/k8s/local/secret.yaml`：本地 k3d 可选 runtime 镜像列表，将 v0.18.2 标记为当前版本。
- `deploy/k8s/prod/secret.yaml`：生产可选 runtime 镜像列表，登记 v0.18.2 的待发布不可变镜像前缀。
- `deploy/k8s/prod/secret.example.yaml`：生产配置模板使用语义版本号示例。

- [ ] **Step 1: 创建最小新变体元数据作为失败用例输入**

使用 `apply_patch` 创建以下两个文件：

```text
# runtime/hermes/hermes-v0.18.2/version.txt
v0.18.2

# runtime/hermes/hermes-v0.18.2/hermes-ref.txt
v2026.7.7.2
```

- [ ] **Step 2: 运行 dry-run，确认现有构建错误地使用产品版本拉取上游**

Run:

```bash

```

Expected: FAIL，退出码为 1；dry-run 实际仍输出 `HERMES_REF=v0.18.2`，证明当前
Makefile 尚未读取 `hermes-ref.txt`。

- [ ] **Step 3: 添加最小 Makefile ref 派生与守卫**

在 `HERMES_VERSION` 后加入：

```make
# HERMES_UPSTREAM_REF 单独记录上游 Git ref；新语义版本变体使用 hermes-ref.txt，
# 历史日期版本没有该文件时回退到 version.txt，避免破坏已有构建入口。
override HERMES_UPSTREAM_REF := $(strip $(shell if [ -f "$(HERMES_VARIANT_DIR)/hermes-ref.txt" ]; then cat "$(HERMES_VARIANT_DIR)/hermes-ref.txt"; else printf '%s' "$(HERMES_VERSION)"; fi))
```

在 `.guard-hermes-version` 的非空版本检查后加入：

```make
	@test -n "$(HERMES_UPSTREAM_REF)" || { echo "Hermes 上游 ref 不能为空: $(HERMES_VARIANT_DIR)/hermes-ref.txt" >&2; exit 1; }
```

将普通 runtime 的两个 Docker build 参数从：

```make
--build-arg "HERMES_REF=$(HERMES_VERSION)"
```

改为：

```make
--build-arg "HERMES_REF=$(HERMES_UPSTREAM_REF)"
```

并将默认变体改为：

```make
HERMES_VARIANT       ?= hermes-v0.18.2
```

- [ ] **Step 4: 验证新旧变体解析均正确**

Run:

```bash
make -n build-hermes-runtime HERMES_VARIANT=hermes-v0.18.2 | grep -- '--build-arg "HERMES_REF=v2026.7.7.2"'
make -n build-hermes-runtime HERMES_VARIANT=hermes-v2026.7.1 | grep -- '--build-arg "HERMES_REF=v2026.7.1"'
make .guard-hermes-version HERMES_VARIANT=hermes-v0.18.2
```

Expected: 两个 grep 均匹配，版本守卫退出码为 0。

- [ ] **Step 5: 提交构建元数据改动**

