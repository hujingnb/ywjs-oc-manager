# Chinese Comments Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 补齐全部人工维护源码中的中文注释，使文件、类型、字段、常量、函数、参数语义和关键代码段都能说明业务意图、边界条件和异常原因。

**Architecture:** 本计划只改注释，不改运行行为、不重排无关代码。按后端核心、业务服务、异步集成、存储包装、前端公共逻辑、前端页面组件分批处理，每批先生成候选清单，再人工补齐高价值中文注释并运行对应验证。

**Tech Stack:** Go 1.x、Gin、pgx/sqlc、Vue 3、TypeScript、Pinia、TanStack Vue Query、Vitest、vue-tsc、Vite。

---

## File Structure

修改范围是人工维护源码：

- `cmd/**/*.go`
- `internal/**/*.go`
- `web/src/**/*.ts`
- `web/src/**/*.vue`

必须排除：

- `internal/store/sqlc/*.go`：文件头含 `Code generated by sqlc. DO NOT EDIT.`
- `web/src/api/generated.ts`：OpenAPI 生成的前端类型。
- `openapi/openapi.yaml`：OpenAPI 生成产物。
- `node_modules`、`dist`、`coverage`、`tmp` 等依赖、构建和临时目录。

注释补齐责任分布：

- 后端文件：包/文件职责、导出和关键非导出类型、结构体字段、接口方法、枚举常量、服务方法、权限/状态/事务/重试/外部调用代码段。
- 后端测试：测试文件职责、fake/mock 的模拟边界、测试用例覆盖的业务场景、关键断言组的原因。
- 前端 TypeScript：模块职责、类型字段、权限/状态格式化、API 请求、store/composable 的数据流和错误边界。
- Vue 组件：页面/组件职责、关键 computed/ref、权限分支、事件处理、数据提交和展示降级逻辑。
